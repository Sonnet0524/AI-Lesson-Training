📊 项目设计：智能问数

项目名称：电力数据智能问答系统
项目目标：通过自然语言对话方式查询电力数据，自动生成数据分析和可视化图表
目标用户：电力公司管理人员、数据分析人员、运维人员
复杂度：⭐⭐⭐

一、业务场景

典型需求

场景1：领导查询
"查一下本月各区域的用电量排名"
场景2：趋势分析
"对比一下成都高新区最近7天的负荷变化趋势"
场景3：异常排查
"昨天半夜负荷突然升高是什么原因？"
场景4：数据对比
"今年一季度和去年同期相比，哪些区域增长最快？"

二、技术架构

Agent流程设计（ReAct模式）

整体流程
用户提问
→
信息提取
→
HTTP调用
→
信息加工
→
智能对话

ReAct执行过程

【Thought】分析用户意图 "用户想查询用电量数据，需要识别时间范围、区域、指标" 【Action】调用信息提取 提取参数：{ "metric": "用电量", "region": "各区域", "time_range": "本月", "operation": "排名" } 【Observation】获取参数成功 【Thought】确定查询方案 "需要查询本月各区域用电量，然后排序" 【Action】调用数据查询API query_power_data(metric="用电量", region="all", time_range="本月") 【Observation】获取数据 [{"region": "高新区", "value": 850}, ...] 【Thought】数据处理和可视化 "数据已获取，需要排序并生成图表" 【Final Answer】返回结果 "本月各区域用电量排名如下： 1. 高新区：850万kWh 2. 天府新区：820万kWh..."

三、灵知平台模块配置

模块1：用户提问

配置

  • ☑ 输入文本
  • ☐ 上传文档
  • ☐ 上传图片

模块2：信息提取

提取字段：
  • metric - 查询指标（用电量/负荷/电量）
  • region - 查询区域
  • time_range - 时间范围
  • operation - 操作类型（排名/对比/趋势）

模块3：HTTP调用

请求体模板：
{ "metric": "{{信息提取.metric}}", "region": "{{信息提取.region}}", "time_range": "{{信息提取.time_range}}", "operation": "{{信息提取.operation}}" }

模块4：智能对话

系统提示词：

你是电力数据分析助手，负责解读查询结果并以清晰的方式呈现给用户。

回复创意性：0.3

聊天上下文：3
