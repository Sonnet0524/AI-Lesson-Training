📅 Day 3：Agent 优化与 API 发布

课时：8 课时（上午 4 课时 + 下午 4 课时 + 晚自习 2 课时）
目标：
  • 掌握高级模块（循环、代码块、多智能体）
  • 完成 Agent 功能优化
  • 发布 API 供前端调用
  • 通过中期检查

上午课程：高级功能

课时 1-2：循环与代码块（80 分钟）

1. 循环模块

功能：用于循环流程，处理数组类型数据
参数：
  • 元素序号：记录第 n 次，{{xxxx.index}}
  • 元素值：某个元素，{{xxxx.item}}
  • 数组长度：{{xxxx.length}}
场景：批量处理文档、遍历查询结果、循环调用 API

2. 代码块模块

# Python 示例 def userFunction(param): input_text = param['input_key'] result = { 'output_key': input_text.upper(), 'length': len(input_text) } return result // JavaScript 示例 function userFunction(param) { var input1 = param['input_key']; var result = {}; result['output_key'] = input1.toUpperCase(); return result; }

课时 3：多智能体联动（40 分钟）

多智能体架构

  • 主智能体：协调调度
  • 子智能体：专门功能

Agent 对话结束模块

关键限制：每个子智能体仅允许包含一个"Agent 对话结束"模块
参数：
  • 展示确认弹窗：开启后弹出提示框
  • 满意，继续执行母 Agent
  • 不满意，重新执行子 Agent

案例：SEO 文章生成系统

主智能体：SEO 文章生成 ├─ 子智能体 1：确认标题 ├─ 子智能体 2：参考文章提炼 ├─ 子智能体 3：收集资料需求 └─ 子智能体 4：主题生成

下午课程：优化与发布

课时 5-6：Agent 优化（80 分钟）

1. 回复质量优化

  • 提示词优化 - 明确角色定位、详细说明任务、提供示例
  • 知识库优化 - 调整相似度阈值、优化召回数、开启重排序
  • 上下文优化 - 设置合适的聊天上下文条数

2. 响应速度优化

  • 减少不必要的模块
  • 降低召回数（如从 20 降到 10）
  • 谨慎使用重排序（消耗较大）
  • 降低回复字数上限

3. 用户体验优化

优化前： "根据查询结果高新区用电量最高是 850MW 其次是天府新区 820MW..." 优化后： "📊 用电查询结果： 1. 高新区：850MW 2. 天府新区：820MW 3. 锦江区：780MW 数据来源：用电管理系统 更新时间：2024-03-01"

课时 7-8：API 发布与测试（80 分钟）

1. 发布流程

  1. 点击右上角"···" → "发布"
  2. 选择"API 服务"标签
  3. 点击"创建链接"

2. API 调用方法

POST https://api.xxx.com/agent/{uuid} Content-Type: application/json Authorization: Bearer {AuthKey} { "query": "用户问题", "context": {} } 响应: { "code": 200, "data": { "answer": "Agent 回复内容", "trace_id": "追踪 ID" } }

3. React 调用示例

const callAgent = async (question) => { const response = await fetch(`${API_URL}/${UUID}`, { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${AUTH_KEY}` }, body: JSON.stringify({ query: question }) }); const result = await response.json(); return result.data.answer; };
