📝 项目设计：智能写作

项目名称：电力公文智能写作助手
项目目标：通过AI辅助快速生成电力行业公文、报告、总结等文档
目标用户：电力公司行政人员、管理人员、技术人员
复杂度：⭐⭐

一、业务场景

典型需求

场景1：工作报告
"帮我写一份月度工作总结，包含安全生产、客户服务、设备运维三个方面"
场景2：会议纪要
"根据会议记录生成会议纪要，突出重点决议和行动计划"
场景3：通知公告
"起草一份关于春季安全大检查的通知"
场景4：方案撰写
"写一份智能电表改造项目实施方案"

二、技术架构

Agent流程设计（多智能体模式）

主智能体协调流程
用户输入需求
→
主智能体
→
类型识别
→
要点提取
内容生成
→
格式优化
→
输出文档

子智能体分工

【主智能体：写作协调器】 分析需求 → 协调各子智能体 【子智能体1：类型识别】 判断文档类型：工作总结/会议纪要/通知公告/方案 【子智能体2：要点提取】 从用户输入中提取关键要点 【子智能体3：内容生成】 根据类型和要点生成内容，调用知识库获取模板 【子智能体4：格式优化】 按照公文格式规范调整，检查标题层级、段落结构

三、灵知平台模块配置

模块流程

用户提问
→
信息提取
→
知识库搜索
→
智能对话
→
代码块格式化

模块1：信息提取

提取字段：
  • doc_type - 文档类型（工作总结/会议纪要/通知公告/实施方案）
  • subject - 主题或标题
  • key_points - 关键要点
  • period - 时间周期
  • audience - 目标受众

模块2：知识库搜索

配置

  • 知识库：公文模板库
  • 召回数：3
  • 相似度阈值：0.5
  • 开启重排序：是

模块3：智能对话（内容生成）

系统提示词要点：
  • 严格遵守电力行业公文格式规范
  • 语言正式、准确、简洁
  • 结构清晰、层次分明
  • 内容专业、数据准确
  • 适当使用行业术语
