上午课程：Vibe Coding 入门与界面开发

课时 1：Vibe Coding 开发模式（40 分钟）

什么是 Vibe Coding？

传统开发：程序员手动编写每一行代码
Vibe Coding：用自然语言描述需求，AI 生成代码，程序员审核调整

Vibe Coding 工作流

Vibe Coding 循环
描述需求
→
AI 生成代码
→
运行测试
→
反馈调整
→
描述需求

有效沟通技巧

✅ 好的描述❌ 不好的描述
具体：描述功能、样式、交互太模糊："做一个界面"
渐进：从简单开始，逐步迭代不具体："改一下那个东西"
明确：说明不想要的和想要的缺少上下文："写个组件"

课时 2-3：使用 Vibe Coding 开发聊天界面（80 分钟）

与 opencode 对话示例

用户：帮我创建一个 React 项目，使用 Vite， 项目名叫 power-chat-app。 opencode：好的，我来帮你创建项目... [执行命令] npm create vite@latest power-chat-app -- --template react --- 用户：创建聊天界面的基本布局，包含头部、 消息区域、输入区域三部分。 opencode：我来创建聊天界面组件... [生成 ChatPage.jsx 和对应 CSS] --- 用户：消息区域需要显示对话消息，每条消息有 用户消息和 AI 消息两种样式。 opencode：我来添加消息显示功能... [创建 MessageItem 组件，添加两种样式]

课时 4：项目结构与代码审查（40 分钟）

代码审查要点

  • 代码结构是否清晰
  • 组件划分是否合理
  • 是否有冗余代码
  • 样式是否统一
  • 是否有潜在 bug
