🎫 项目设计：工单处理

项目名称：电力工单智能处理系统
项目目标：自动分析工单内容，智能分配处理部门，生成处理建议
目标用户：电力公司客服人员、工单处理人员、管理人员
复杂度：⭐⭐⭐⭐

一、业务场景

典型需求

场景1：故障报修
用户来电："我们家停电了，地址是XX小区XX栋XX号"
→ 自动识别：故障报修、地址提取、派发给运维班组
场景2：业务咨询
用户咨询："我想申请新装电表，需要什么材料？"
→ 自动识别：业务咨询、回复标准答案
场景3：投诉建议
用户投诉："上个月电费突然变高很多，我要投诉"
→ 自动识别：投诉类型、生成处理方案、升级优先级
场景4：信息查询
用户查询："我想查一下上个月的用电量"
→ 自动识别：信息查询、调用系统接口、返回数据

二、技术架构

Agent流程设计（ReAct + 循环模式）

工单处理流程
用户提交工单
→
分类识别
→
信息提取
查询责任部门
→
知识库检索
→
生成处理建议

ReAct执行过程示例

【Thought】分析工单类型 "用户反馈停电问题，可能是故障报修类型" 【Action】调用分类识别 分类结果：{ "type": "故障报修", "confidence": 0.95 } 【Observation】分类成功 【Thought】提取关键信息 "需要提取地址、联系方式、故障描述等关键信息" 【Action】调用信息提取 提取结果：{ "address": "XX小区XX栋XX号", "phone": "138****1234", "description": "家里突然停电", "urgency": "高" } 【Thought】查询处理部门 "故障报修需要分配给运维班组" 【Action】调用HTTP接口查询责任部门 查询结果：{ "department": "城东运维班", "contact": "李工" } 【Final Answer】返回完整处理结果 { "工单编号": "GD20240303001", "分类": "故障报修-停电", "处理部门": "城东运维班", "处理建议": "1. 核实停电范围 2. 安排现场勘查..." }

三、灵知平台模块配置

模块流程

用户提问
→
信息分类
→
信息提取
→
HTTP调用
知识库搜索
→
智能对话

模块1：信息分类

分类选项：
  • 故障报修：停电、线路故障、设备故障等
  • 业务咨询：新装、增容、过户、缴费等
  • 投诉建议：服务投诉、电费争议、质量投诉等
  • 信息查询：电量查询、账单查询、进度查询等
  • 其他

模块2：信息提取

提取字段

  • address - 地址（详细到门牌号）
  • contact - 联系电话
  • description - 问题描述
  • time - 发生时间
  • urgency - 紧急程度（高/中/低）

模块3：HTTP调用（查询责任部门）

请求体模板：
{ "type": "{{信息分类.type}}", "address": "{{信息提取.address}}", "urgency": "{{信息提取.urgency}}" }
