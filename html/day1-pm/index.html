Agent本质与RPAR框架

Agent的本质：从"动口"到"动手"

💬

传统大模型应用

  • ❌ 只给建议，不执行
  • ❌ 单轮对话
  • ❌ 告诉你"怎么做"
  • 💡 价值：信息
VS
🤖

Agent

  • ✅ 直接执行，完成任务
  • ✅ 多轮自主交互
  • ✅ 直接"帮你做完"
  • 🎯 价值：结果
核心洞察

"Agent的本质是'执行'，特别是'自主执行'——从'顾问'到'助手'"

Workflow vs Agentic：从"固定"到"自主"

🚄

Workflow（高铁）

A → B → C
  • ✅ 速度快，按步骤执行
  • ❌ 路线固定
  • ❌ 走错就停止
VS
🚗

Agentic（自驾）

起点 ├→ B →┐ └→ C →┘ 终点
  • ✅ 可根据路况调整
  • ✅ 自主规划路线
  • ✅ 实时灵活应变

02

RPAR实例与Acting详解

40分钟

RPAR的裁剪：不同场景的选择

ReAct: R-A-R 简化版

Reasoning → Acting → Reasoning
特点
  • ✓ 简单直接，边想边做
  • ✓ 适用于简单查询类任务
  • ✗ 缺乏显式规划和反思
示例：查询天气
思考→调用天气API→回答用户

Plan-and-Execute: R-P-A 高效版

Reasoning → Planning → Acting(List)
特点
  • ✓ 先规划后批量执行
  • ✓ 适用于步骤明确的任务
  • ✗ 缺乏反思调整能力
示例：订机票
规划(查航班→选航班→预订)→批量执行

完整RPAR: R-P-A-R 完全体

Reasoning → Planning → Acting → Reflection
特点
  • ✓ 完整四环节，支持试错
  • ✓ 适用于复杂探索性任务
  • ✗ 复杂度最高
示例：投资分析
多轮循环：查询→分析→反思→再查询...

范式选择决策树

任务复杂度?
简单明确
→ ReAct
复杂
步骤明确?
是
→ Plan-and-Execute
否
→ 完整RPAR
