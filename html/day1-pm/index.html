Agentic AI 详解

与Autonomous AI的区别

Agentic AI 是一种能够自主感知环境、做出决策并执行行动的AI系统。与传统的Autonomous AI不同，Agentic AI强调"主动性"和"目标导向性"。

核心特征
  • 自主性：能够独立做出决策，无需人工干预
  • 主动性：主动探索解决问题的方法，而非被动响应
  • 适应性：根据环境变化动态调整策略
  • 工具使用：能够调用外部工具和资源完成复杂任务
与Autonomous AI的区别
维度 Autonomous AI Agentic AI
决策方式 基于规则/预设 基于推理和学习
交互方式 被动响应 主动规划
目标达成 固定流程 动态调整

Workflow vs Agentic：从"固定"到"自主"

🧠

Reasoning - 思考环节

AI理解用户意图、分析当前状况、识别约束条件和潜在风险。

  • 意图识别：理解用户的真实需求
  • 上下文理解：分析对话历史和背景信息
  • 约束分析：识别时间、资源等限制
  • 风险评估：预判可能的问题
📋

Planning - 规划环节

基于思考结果，制定详细的执行计划，包括步骤分解和优先级排序。

  • 任务分解：将复杂任务拆分为可执行子任务
  • 优先级排序：确定执行顺序
  • 资源评估：评估所需工具和数据
  • 路径规划：选择最优执行路径
⚡

Acting - 执行环节

调用各种工具和资源，执行规划中的具体任务。

  • 工具选择：根据任务选择合适工具
  • 参数准备：准备正确的输入参数
  • 执行调用：发起工具调用
  • 结果获取：接收并解析执行结果
🔄

Reflection - 反思环节

评估执行结果，识别问题，调整策略，决定是否继续循环。

  • 结果评估：检查结果是否符合预期
  • 错误识别：发现并定位问题
  • 策略调整：修改后续执行计划
  • 知识沉淀：总结经验教训

ReAct 详解

论文来源与核心思想

ReAct（Reasoning + Acting）是普林斯顿大学和Google Research在2022年提出的方法，核心思想是让AI交替进行推理和行动。

论文信息
  • 论文标题：ReAct: Synergizing Reasoning and Acting in Language Models
  • 发表时间：2022年10月
  • 核心作者：Shunyu Yao（普林斯顿）, Jeffrey Zhao（Google）等
  • 核心思想：推理（Reasoning）和行动（Acting）应该相互促进
完整流程示例
ReAct循环伪代码 
def react_loop(user_query, max_iterations=10):
    thought_history = []
    action_history = []
    
    for i in range(max_iterations):
        # 1. Reasoning: 基于历史和当前状态思考
        thought = llm.reason(
            query=user_query,
            history=thought_history,
            actions=action_history
        )
        thought_history.append(thought)
        
        # 2. Acting: 执行工具调用
        action = parse_action(thought)
        observation = execute_tool(action)
        action_history.append({"action": action, "observation": observation})
        
        # 3. 检查是否完成
        if is_complete(thought, observation):
            return generate_answer(thought_history, action_history)
    
    return "达到最大迭代次数"

Python API调用示例 Python 
import requests
from typing import Dict, Any

class PowerSystemAPI:
    """电力系统API调用工具"""
    
    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
    
    def query_investment_data(
        self, 
        region: str, 
        start_date: str, 
        end_date: str,
        project_type: str = "全部"
    ) -> Dict[str, Any]:
        """
        查询投资项目数据
        
        Args:
            region: 查询区域，如'成都高新区'
            start_date: 开始日期，格式YYYY-MM-DD
            end_date: 结束日期，格式YYYY-MM-DD
            project_type: 项目类型筛选
        
        Returns:
            包含项目列表、投资总额等信息的字典
        """
        url = f"{self.base_url}/api/v1/investments/query"
        params = {
            "region": region,
            "start": start_date,
            "end": end_date,
            "type": project_type
        }
        
        try:
            response = requests.get(url, headers=self.headers, params=params)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            return {"error": str(e), "code": "API_ERROR"}

MCP (Model Context Protocol)

模型上下文协议详解

MCP是一种标准化协议，让AI模型能够理解工具的用途、参数和返回值。它定义了一套JSON Schema，使得工具具备"自描述"能力。

MCP的核心价值
  • 第一层价值 - 描述标准化：统一的JSON格式，让AI直接理解
  • 第二层价值 - 生态标准化：跨平台互通，一次开发多处使用
MCP字段说明
字段 类型 说明
name string 工具名称，驼峰命名
description string 功能描述，让AI理解用途
parameters object 输入参数定义（JSON Schema）
returns object 返回结果定义
errors object 错误码定义

你是一位智能工单处理专家。你的任务是自动识别工单类型、智能分配给合适的处理人、跟踪处理进度并提供自动回复。

## 处理流程
1. 接收工单内容，分析工单描述和附件
2. 使用classify_workorder工具识别工单类型（故障/咨询/投诉/建议）
3. 根据工单类型和紧急程度，使用find_best_handler工具找到最合适的处理人
4. 使用generate_reply工具生成专业的自动回复
5. 使用create_workorder工具创建正式工单并分配
6. 持续跟踪处理进度，超时自动升级

## 工单优先级定义
- urgent：紧急故障，影响业务运行
- high：重要问题，需要尽快处理
- medium：一般问题，按正常流程处理
- low：咨询建议类，可延后处理

## 分配原则
- 根据工单类型匹配专业技能
- 考虑处理人当前工作负荷
- 优先分配给历史处理效率高的员工
