# 项目设计：智能写作

## 项目概述

**项目名称**：电力公文智能写作助手

**项目目标**：通过AI辅助快速生成电力行业公文、报告、总结等文档

**目标用户**：电力公司行政人员、管理人员、技术人员

**复杂度**：⭐⭐

---

## 一、业务场景

### 典型需求

```
场景1：工作报告
"帮我写一份月度工作总结，包含安全生产、客户服务、设备运维三个方面"

场景2：会议纪要
"根据会议记录生成会议纪要，突出重点决议和行动计划"

场景3：通知公告
"起草一份关于春季安全大检查的通知"

场景4：方案撰写
"写一份智能电表改造项目实施方案"
```

### 用户痛点

- 传统方式：手写文档耗时长、格式规范难掌握、内容质量参差不齐
- 智能写作：输入要点 → 自动生成 → 格式规范 → 可快速修改

## 二、技术架构

### 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端应用层                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  需求输入   │  │  文档编辑   │  │  模板选择   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
                           ↓ API调用
┌─────────────────────────────────────────────────────────────┐
│                      Agent层（灵知平台）                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│  │ 类型识别 │→│ 要点提取 │→│ 内容生成 │→│ 格式优化 │       │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘       │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                      知识层                                  │
│     公文模板库 │ 范文库 │ 行业规范 │ 专业术语                │
└─────────────────────────────────────────────────────────────┘
```

### Agent流程设计（多智能体模式）

```
用户输入需求
     ↓
【主智能体：写作协调器】
分析需求 → 识别文档类型
     ↓
【子智能体1：类型识别】
判断文档类型：工作总结/会议纪要/通知公告/方案
     ↓
【子智能体2：要点提取】
从用户输入中提取关键要点
输出：{
  "type": "工作总结",
  "period": "月度",
  "aspects": ["安全生产", "客户服务", "设备运维"],
  "tone": "正式"
}
     ↓
【子智能体3：内容生成】
根据类型和要点生成内容
调用知识库获取模板和范文
     ↓
【子智能体4：格式优化】
按照公文格式规范调整
检查标题层级、段落结构
     ↓
【返回主智能体】
输出最终文档
```

#### 2. 信息提取模块

```
模块名称：写作需求提取
提示词：
请从用户需求中提取以下信息：
- doc_type：文档类型（工作总结/会议纪要/通知公告/实施方案/其他）
- subject：主题或标题
- key_points：关键要点（数组）
- period：时间周期（如适用）
- audience：目标受众（如适用）

输出JSON格式，如：
{
  "doc_type": "工作总结",
  "subject": "月度工作总结",
  "key_points": ["安全生产", "客户服务", "设备运维"],
  "period": "2024年3月",
  "audience": "公司领导"
}

提取字段配置：
┌─────────────────────────────────────┐
│ 字段key: doc_type                   │
│ 描述: 文档类型                      │
│ 必填: 是                            │
├─────────────────────────────────────┤
│ 字段key: subject                    │
│ 描述: 文档主题                      │
│ 必填: 是                            │
├─────────────────────────────────────┤
│ 字段key: key_points                 │
│ 描述: 关键要点                      │
│ 必填: 否                            │
├─────────────────────────────────────┤
│ 字段key: period                     │
│ 描述: 时间周期                      │
│ 必填: 否                            │
└─────────────────────────────────────┘

回复创意性：0
```

#### 3. 知识库搜索模块

```
模块名称：模板检索
知识库：公文模板库
配置：
- 召回数：3
- 相似度阈值：0.5
- 开启重排序：是

检索变量：{{信息提取.doc_type}}

输出：
- 相关模板 → 智能对话模块
```

#### 4. 智能对话模块

```
模块名称：内容生成
系统提示词：
你是国网四川电力的公文写作助手，擅长撰写各类电力行业公文。

当前任务：
- 文档类型：{{信息提取.doc_type}}
- 文档主题：{{信息提取.subject}}
- 关键要点：{{信息提取.key_points}}
- 时间周期：{{信息提取.period}}

参考模板：
{{知识库搜索结果}}

写作要求：
1. 严格遵守电力行业公文格式规范
2. 语言正式、准确、简洁
3. 结构清晰、层次分明
4. 内容专业、数据准确
5. 适当使用行业术语

请生成完整的文档内容。

回复创意性：0.7
聊天上下文：2
回复字数上限：2000
```

#### 5. 代码块模块

```
模块名称：格式化输出
类型：JavaScript

function userFunction(param) {
  const content = param['content'];
  
  // 格式化为Markdown
  let formatted = content;
  
  // 添加标题标识
  if (!formatted.startsWith('#')) {
    formatted = '# ' + param['subject'] + '\n\n' + formatted;
  }
  
  // 统计字数
  const wordCount = formatted.length;
  
  return {
    document: formatted,
    word_count: wordCount,
    generated_at: new Date().toISOString()
  };
}

输出：
- document：格式化文档
- word_count：字数统计
```
