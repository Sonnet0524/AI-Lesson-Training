## 上午课程：Agent设计原理与模块详解

### 课时1：规划页面与模块架构（40分钟）

#### 学习目标
- 熟悉规划页面的三大区域
- 理解节点类型与连接原则
- 掌握模块的三部分结构

#### 课程内容

**1. 规划页面概览（15分钟）**

**界面三大区域**：
```
┌─────────────────────────────────────────────────────────────┐
│                      功能区                                  │
│  [导入/导出] [试运行]           [分享] [删除] [保存] [发布]  │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│ 模块区   │                  画布区                           │
│          │                                                  │
│ - 系统模块│    （拖拽组件、连线编排的区域）                   │
│ - 智能体 │                                                  │
│ - 工具   │                                                  │
│ - MCP   │                                                  │
│          │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

**模块区4类组件**：
- **系统模块**：用户提问、智能对话、信息分类、信息提取、信息加工、确定回复、知识库搜索等
- **智能体**：调用已上线的其他智能体（子智能体）
- **工具**：官方工具 + 自定义工具
- **MCP**：连接外部服务（SSE方式）

**节点类型与连接原则**：
| 颜色 | 类型 | 连接规则 |
|------|------|----------|
| 黄色 | 布尔型 | 只能连布尔型 |
| 蓝色 | 字符串 | 文本/图片/文档信息 |
| 紫色 | 知识库结果 | 仅连知识库相关 |
| 红色 | 任意类型 | 仅用于循环模块 |

**关键原则**：同类型节点互相连接，不同类型不可连接

---

**2. 模块结构详解（15分钟）**

每个模块由三部分组成：
```
┌─────────────────────────┐
│  模块输入区（左上角）    │  ← 连接节点将信息输入
│  - 联动激活              │
│  - 任一激活              │
│  - 信息输入              │
├─────────────────────────┤
│  模块设置区（中间）      │  ← 设置参数、提示词
│  - 聊天上下文            │
│  - 模型选择              │
│  - 系统提示词            │
│  - 回复创意性            │
├─────────────────────────┤
│  模块输出区（右下角）    │  ← 执行后输出信息
│  - 回复结束              │
│  - 回复内容              │
│  - 模块运行结束          │
└─────────────────────────┘
```

### 课时2-3：核心模块详解（80分钟）

#### 学习目标
- 掌握5个核心模块的功能与参数
- 能够根据场景选择合适的模块
- 理解模块间的配合使用

#### 课程内容

**1. 用户提问模块（15分钟）**

**功能**：获取用户提问/选择内容，传输给其他模块

**核心参数**：
- 输入文本（默认打开）
- 上传文档（最多9个，每个50MB）
- 上传图片（最多9个，单个30M）
- 文档审查（配合文档审查模块）
- 添加按钮（明确执行节点）

**输出**：
- 文本信息、文档信息、图片信息
- 未点击按钮、模块运行结束

**使用场景**：
- 所有Agent的入口模块
- 需要用户上传文件的场景
- 需要按钮交互的场景

---

**2. 智能对话模块（20分钟）**

**功能**：通过大语言模型处理并回复用户

**核心参数**：
| 参数 | 建议值 | 说明 |
|------|--------|------|
| 聊天上下文 | 0-6条 | 贴近用户意图 |
| 回复创意性 | 0-1 | 0严谨，1发散（知识库场景建议0） |
| 核采样TOP_P | 0-1 | 控制多样性，一般与创意性只设一个 |
| 回复字数上限 | 100-8192 | 根据场景设定 |
| 回复对用户可见 | 是/否 | 中间流程可设为否 |

**输入**：
- 信息输入（文本）
- 图片输入
- 知识库搜索结果

**输出**：
- 回复结束（是/否，可作触发器）
- 回复内容
- 模块运行结束

**使用场景**：
- 所有需要AI回复的场景
- 角色扮演
- 内容生成

**案例**：
```
场景：公文写作助手
系统提示词：你是一名专业的公文写作助手，擅长撰写各类公文...
回复创意性：0.3（需要一定创意但保持规范）
聊天上下文：3条
```

---

**3. 信息分类模块（15分钟）**

**功能**：将用户问题分类，针对不同类型执行不同操作

**核心参数**：
- 提示词：说明分类任务
- 标签：设置分类类别（如"咨询""投诉""建议""其他"）

**输出**：
- 标签节点（布尔型）：符合该分类输出True
- 分类结果（JSON格式）
- 模块运行结束

**使用场景**：
- 工单分类
- 问题类型识别
- 多流程分支

**案例：工单处理Agent**
```
标签设置：
- 故障报修 → 连接故障处理流程
- 业务咨询 → 连接咨询回复流程
- 投诉建议 → 连接投诉处理流程
- 其他 → 引导用户重新描述

提示词：请判断用户问题属于哪种类型...
```

---

**4. 信息提取模块（15分钟）**

**功能**：从输入信息中提取目标信息（关键词、SQL等）

**核心参数**：
- 提示词：说明提取任务
- 提取字段：配置字段key、描述、是否必填

**输出**：
- 完全提取/必填完全提取/提取缺失（布尔型）
- 全部提取结果（JSON字符串）

**使用场景**：
- 提取搜索关键词
- 提取结构化数据
- 配合HTTP调用使用

**案例：会议室预定**
```
用户输入："预约天空之门，下午四点，预约人小助手"

提取字段配置：
- 字段key: room_name, 描述: 会议室名称, 必填: 是
- 字段key: time, 描述: 预约时间, 必填: 是
- 字段key: person, 描述: 预约人姓名, 必填: 是

提取结果：{"room_name":"天空之门","time":"16:00","person":"小助手"}
```

---

**5. 知识库搜索模块（15分钟）**

**功能**：在知识库中搜索相关问题与解答

**核心参数**：
| 参数 | 范围 | 建议 |
|------|------|------|
| 相似度阈值 | 0-1 | 0.6-0.9（越高越严格） |
| 向量相似度权重 | 0-1 | 默认1，有关键词时降低 |
| 召回数 | 0-100 | 建议5-10 |
| 是否开启重排序 | 是/否 | 提高精确度但降低速度 |
| 重排序召回数 | 0-20 | 建议5-10 |

**输出**：
- 未搜索到相关知识（布尔型）
- 搜索到相关知识（布尔型）
- 知识库搜索结果（数组格式）

**使用场景**：
- 知识库问答
- 文档检索
- 客服问答

**案例：电力知识问答**
```
关联知识库：电力安规知识库、设备手册知识库
相似度阈值：0.7
召回数：5
重排序：开启

流程：
用户提问 → 知识库搜索 → 
  ├─ 搜索到 → 智能对话（结合搜索结果回复）
  └─ 未搜索到 → 确定回复（"抱歉，暂时无法回答"）
```

### 课时4：模块组合与流程设计（40分钟）

#### 学习目标
- 掌握多模块组合的设计方法
- 能够设计完整的Agent流程
- 了解常见场景的编排模式

#### 课程内容

**1. 常见编排模式（20分钟）**

**模式1：简单问答**
```
用户提问 → 智能对话 → 回复用户
```

**模式2：知识库问答**
```
用户提问 → 知识库搜索 → 
  ├─ 搜索到 → 智能对话（结合知识库） → 回复
  └─ 未搜索到 → 确定回复（默认回答）
```

**模式3：分类处理**
```
用户提问 → 信息分类 → 
  ├─ 类型A → 处理流程A → 回复
  ├─ 类型B → 处理流程B → 回复
  └─ 其他 → 引导重新描述
```

**模式4：提取+API调用**
```
用户提问 → 信息提取 → HTTP调用 → 信息加工 → 回复
```

**模式5：多轮对话**
```
用户提问 → 信息提取 → 
  ├─ 提取完整 → 处理并回复
  └─ 提取缺失 → 确定回复（引导补充信息）
```

---

**2. 设计技巧（15分钟）**

**技巧1：按信息流转顺序排布模块**
- 从左到右、从上到下
- 先规划再连线

**技巧2：持续调试优化**
- 使用"试运行"功能
- 查看每个模块的执行时间
- 调整参数提升效果

**技巧3：注意必填信息**
- 每个模块的必填参数先设置
- 选填参数根据流程需要设置

**技巧4：用户交互设计**
- 设计对话引导内容
- 考虑用户未按要求的回复
- 设置问题建议（最多6个）

## 下午课程：分组开发实战

### 课时5-6：Agent搭建（80分钟）

### B. 参数建议值

| 场景 | 回复创意性 | 相似度阈值 | 召回数 |
|------|------------|------------|--------|
| 知识库问答 | 0 | 0.7-0.8 | 5-10 |
| 创作生成 | 0.6-0.8 | - | - |
| 工单分类 | 0 | - | - |
| 信息提取 | 0 | - | - |

### C. 常用提示词模板

**信息分类**：
```
请判断用户问题属于哪种类型，可选类型：咨询、投诉、建议、其他。
只输出类型名称，不要输出其他内容。
```

**信息提取**：
```
请从用户输入中提取以下信息：
- 会议室名称
- 预约时间
- 预约人姓名

以JSON格式输出，如：{"room":"xxx","time":"xxx","person":"xxx"}
```

**知识库问答**：
```
请根据以下知识库内容回答用户问题：
{{知识库搜索结果}}

如果知识库内容与问题无关，请回答"抱歉，暂时无法回答这个问题"。
```
