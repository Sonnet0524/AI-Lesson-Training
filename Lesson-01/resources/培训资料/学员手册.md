## 第一章 培训概述

### 1.1 培训目标

**总体目标**：
培养具备Agentic AI开发能力的电力行业AI种子团队，能够独立完成从需求分析到系统交付的端到端开发。

**具体目标**：
- ✅ 理解Agentic AI核心概念与应用场景
- ✅ 掌握ReAct模式与MCP协议
- ✅ 熟练使用灵知平台开发Agent应用
- ✅ 掌握React前端开发基础
- ✅ 能够使用Vibe Coding工具提升开发效率
- ✅ 完成一个完整的项目交付

### 1.2 课程安排

**培训时长**：5天（Day 1-5）+ 3次预授课（Pre-Lesson）

#### Pre-Lesson 预授课（直播）

| 次数 | 时间 | 主题 | 学习目标 |
|------|------|------|----------|
| 第1次 | Day1前3天 | 灵知平台入门 | 完成账号注册，掌握基本操作 |
| 第2次 | Day1前2天 | 核心模块与编排 | 掌握5个核心模块，学会常见编排模式 |
| 第3次 | Day1前1天 | Vibe Coding环境准备 | 完成环境配置，创建第一个React项目 |

#### Day 1-5 正式培训

| 天数 | 主题 | 核心内容 | 实操项目 |
|------|------|----------|----------|
| Day 1 | AI趋势与Agentic AI | ReAct模式、MCP协议、Agentic AI概念 | ReAct模式理解与案例分析 |
| Day 2 | Agent开发实战 | 灵知平台深度使用、项目选题开发 | 开始项目Agent开发 |
| Day 3 | Agent优化与发布 | Agent调优、API发布、测试 | 完成Agent开发并发布API |
| Day 4 | React前端开发 | Vibe Coding、React组件、API对接 | 开发前端界面 |
| Day 5 | 项目展示与总结 | 项目演示、经验分享、成果验收 | 项目展示与总结 |

### 1.3 项目选题

**三个项目方向**：

#### 项目一：智能问数
```
业务场景：自然语言查询电力数据
核心功能：
- 自然语言理解用户查询意图
- 查询用电量、负荷、功率等数据
- 数据可视化展示

技术架构：
前端：React + ECharts图表
Agent：用户提问 → 信息提取 → HTTP调用 → 智能对话
核心模块：信息提取、HTTP调用、智能对话
```

#### 项目二：智能写作
```
业务场景：公文报告智能生成
核心功能：
- 理解写作需求和类型
- 基于知识库生成报告内容
- 提供模板和参考

技术架构：
前端：React + 富文本编辑器
Agent：用户提问 → 信息分类 → 知识库搜索 → 智能对话
核心模块：信息分类、知识库搜索、智能对话
```

#### 项目三：工单处理
```
业务场景：工单智能分类与处理
核心功能：
- 工单自动分类
- 提供处理建议
- 知识库检索相关案例

技术架构：
前端：React + 列表组件
Agent：用户提问 → 信息分类 → 多分支处理
核心模块：信息分类、知识库搜索、确定回复
```

```

步骤4：确认勾选 "Add to PATH"

# 方式2：使用Homebrew
brew install node
```

```bash
# 验证Node.js
node -v
# 预期输出：v18.x.x 或更高

# 验证npm
npm -v
# 预期输出：9.x.x 或更高
```

#### 2.1.3 配置npm镜像

```bash
# 设置淘宝镜像（提升下载速度）
npm config set registry https://registry.npmmirror.com

# 验证配置
npm config get registry
# 预期输出：https://registry.npmmirror.com
```

## 第三章 灵知平台使用手册

### 3.1 平台概述

#### 3.1.1 平台定位

灵知平台是一个低代码Agent开发平台，支持可视化编排AI应用。

**核心能力**：
- 🤖 智能对话
- 📚 知识库检索
- 🏷️ 信息分类
- 📋 信息提取
- 🔌 工具调用

#### 3.1.2 界面布局

```
┌─────────────────────────────────────────────────────┐
│                   灵知平台界面                        │
├──────────┬────────────────────────┬─────────────────┤
│          │                        │                 │
│  模块区  │        画布区          │    功能区       │
│  (左侧)  │        (中央)          │    (上方)       │
│          │                        │                 │
│ - 系统   │  拖拽模块到画布        │  - 导入/导出    │
│ - 智能体 │  连接模块节点          │  - 试运行       │
│ - 工具   │  配置模块参数          │  - 发布/分享    │
│ - MCP    │  测试运行效果          │                 │
│          │                        │                 │
└──────────┴────────────────────────┴─────────────────┘
```

### 3.2 核心模块

#### 3.2.1 用户提问模块

**功能**：获取用户输入，作为Agent入口

**参数配置**：
```
必填项：无
可选项：
- 默认问题：预设用户问题
- 输入提示：引导用户输入
```

**输出**：用户输入的文本内容

#### 3.2.2 智能对话模块

**功能**：调用大语言模型生成回复

**参数配置**：

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 提示词 | 指导模型回复 | 根据场景定制 |
| 回复创意性 | 0-1，越高越有创意 | 知识库问答：0<br/>创作生成：0.6-0.8 |
| 聊天上下文 | 记忆对话轮数 | 3-5轮 |
| 回复字数 | 最大字数限制 | 500-2000字 |

**节点类型**：
```
输入节点：
- 🟡 黄色：布尔型（条件判断）
- 🔵 蓝色：字符串（文本输入）

输出节点：
- 🔵 蓝色：字符串（回复文本）
```

#### 3.2.3 信息分类模块

**功能**：对用户问题进行分类

**参数配置**：
```
提示词示例：
请判断用户问题属于哪种类型，可选类型：故障报修、业务咨询、投诉建议、其他。

各类型说明：
- 故障报修：电力设备故障相关
- 业务咨询：业务办理咨询
- 投诉建议：投诉或建议反馈

只输出类型名称，不要输出其他内容。
```

**输出**：分类结果（字符串）

#### 3.2.4 信息提取模块

**功能**：从文本中提取结构化信息

**参数配置**：
```
提示词示例：
请从用户输入中提取以下信息：
- 查询对象：要查询的设备或区域
- 时间范围：查询的时间段
- 数据类型：用电量/负荷/功率

以JSON格式输出：
{
  "查询对象": "xxx",
  "时间范围": "xxx",
  "数据类型": "xxx"
}
```

**输出**：JSON格式的结构化数据

#### 3.2.5 知识库搜索模块

**功能**：从知识库检索相关内容

**参数配置**：

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 知识库 | 选择知识库 | 根据场景选择 |
| 相似度阈值 | 0-1，越高越严格 | 精确问答：0.8-0.9<br/>一般问答：0.7-0.8 |
| 召回数 | 返回结果数量 | 3-10条 |
| 重排序 | 是否重新排序 | 建议开启 |

**输出**：知识库检索结果

### 3.3 编排模式

#### 3.3.1 简单问答模式

```
用户提问 → 智能对话 → 结束

适用场景：简单对话、通用问答
```

#### 3.3.2 知识库问答模式

```
用户提问 → 知识库搜索 → 智能对话 → 结束

适用场景：基于知识库的专业问答
关键：智能对话提示词包含知识库结果
```

#### 3.3.3 分类处理模式

```
                    ┌─ 智能对话 → 结束
                    │
用户提问 → 信息分类 ─┼─ 知识库搜索 → 智能对话 → 结束
                    │
                    └─ 确定回复 → 结束

适用场景：多分支处理，如工单分类
```

#### 3.3.4 提取+API调用模式

```
用户提问 → 信息提取 → HTTP调用 → 信息加工 → 智能对话 → 结束

适用场景：需要调用外部API，如智能问数
关键：信息提取为API准备参数
```

### 3.4 API发布

#### 3.4.1 发布步骤

```
步骤1：在灵知平台设计好Agent流程
步骤2：点击右上角"发布"
步骤3：选择"API服务"
步骤4：创建链接获取凭证
步骤5：记录Uuid、AuthKey、AuthSecret
```

#### 3.4.2 API凭证

| 信息 | 说明 |
|------|------|
| Uuid | API唯一标识符 |
| AuthKey | 验证密钥 |
| AuthSecret | 安全密钥 |

#### 3.4.3 调用示例

```javascript
// React前端调用灵知平台Agent API
const callAgent = async (question) => {
  const response = await fetch('https://api.xxx.com/agent/{uuid}', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer {AuthKey}'
    },
    body: JSON.stringify({
      query: question,
      context: {}
    })
  });
  
  const result = await response.json();
  return result.data.answer;
};
```

## 第五章 Git与Gitee使用指南

### 5.1 Git基础操作

#### 5.1.1 克隆仓库

```bash
# SSH方式（推荐）
git clone git@gitee.com:用户名/仓库名.git

# HTTPS方式
git clone https://gitee.com/用户名/仓库名.git
```

#### 5.1.2 日常操作

```bash
# 查看状态
git status

# 添加文件
git add .              # 添加所有文件
git add filename.txt   # 添加指定文件

# 提交更改
git commit -m "提交说明"

# 推送到远程
git push origin master

# 拉取更新
git pull origin master
```

#### 5.1.3 查看历史

```bash
# 查看提交历史
git log

# 查看简化历史
git log --oneline

# 查看图形化历史
git log --graph --oneline --all
```

### 5.2 Gitee协作流程

#### 5.2.1 创建Pull Request

```bash
# 1. 创建功能分支
git checkout -b feature-new-function

# 2. 开发并提交
git add .
git commit -m "feat: 实现新功能"
git push origin feature-new-function

# 3. 在Gitee创建Pull Request
#    访问仓库页面 -> Pull Request -> 新建Pull Request

# 4. 等待审查通过

# 5. 合并后删除分支
git checkout master
git pull origin master
git branch -d feature-new-function
```

#### 5.2.2 代码审查要点

```
代码质量：
□ 代码可读性
□ 命名规范
□ 注释完整性

功能实现：
□ 功能完整性
□ 逻辑正确性
□ 异常处理

```

他人的修改
>>>>>>> branch-name

# 3. 手动解决冲突，保留正确的代码

# 4. 标记冲突已解决

## 第七章 常见问题FAQ

### 7.1 环境配置问题

#### Q1：Node.js安装后命令提示"不是内部或外部命令"

**解决方案**：
```
1. 检查是否正确安装
2. 重启命令行窗口
3. 检查环境变量是否配置：
   - Windows：系统属性 -> 高级 -> 环境变量
   - 确认Node.js路径在PATH中
```

#### Q2：npm install速度很慢

**解决方案**：
```bash
# 配置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 或使用cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

#### Q3：SSH连接Gitee失败

**解决方案**：
```bash
# 检查SSH密钥是否正确生成
ls ~/.ssh

# 查看公钥内容
cat ~/.ssh/id_rsa.pub

# 确认已添加到Gitee
# 登录Gitee -> 设置 -> SSH公钥

# 测试连接
ssh -T git@gitee.com
```

### 7.2 灵知平台问题

#### Q4：模块连接不上

**解决方案**：
```
检查节点类型：
- 🟡 黄色只能连黄色（布尔型）
- 🔵 蓝色只能连蓝色（字符串）
- 🟣 紫色只能连紫色（知识库结果）
- 🔴 红色可连任何类型

确保同类型节点才能连接
```

#### Q5：知识库搜索返回空结果

**解决方案**：
```
1. 检查知识库是否有数据
2. 降低相似度阈值（如从0.8降到0.6）
3. 增加召回数（如从3增加到10）
4. 检查用户问题是否与知识库内容相关
```

#### Q6：智能对话回复质量差

**解决方案**：
```
1. 优化提示词，更明确指导模型
2. 调整回复创意性参数
   - 需要准确回答：设为0
   - 需要创造性：设为0.6-0.8
3. 增加聊天上下文轮数（3-5轮）
4. 提供示例对话
```

### 7.3 Git协作问题

#### Q7：推送被拒绝

**解决方案**：
```bash
# 原因：远程有新提交
git push origin master
# ! [rejected]

# 解决：先拉取再推送
git pull origin master
# 解决冲突（如有）
git push origin master
```

#### Q8：如何撤销错误的提交

**解决方案**：
```bash
# 撤销最后一次提交（保留修改）
git reset --soft HEAD~1

# 撤销最后一次提交（丢弃修改）
git reset --hard HEAD~1

# 如果已推送，创建反向提交
git revert <commit-hash>
```

#### Q9：文件冲突如何解决

**解决方案**：
```bash
# 1. 拉取代码发现冲突
git pull origin master

# 2. 打开冲突文件
# 查看冲突标记并手动修改

# 3. 标记已解决
git add 冲突文件
git commit -m "chore: 解决合并冲突"
```

### 7.4 项目开发问题

#### Q10：前端无法调用Agent API

**解决方案**：
```
1. 检查API凭证是否正确
2. 检查网络连接
3. 查看浏览器控制台错误信息
4. 确认API已发布
5. 检查跨域配置（CORS）
```

#### Q11：React项目启动失败

**解决方案**：
```bash
# 删除依赖重新安装
rm -rf node_modules
npm install

# 清除缓存
npm cache clean --force

# 检查端口占用
# Windows
netstat -ano | findstr :5173
# macOS/Linux
lsof -i :5173
```

#### Q12：组件样式不生效

**解决方案**：
```
1. 检查CSS类名是否正确
2. 确认样式文件已导入
3. 检查样式优先级
4. 使用浏览器开发者工具检查元素
```
