## 📝 文件命名规范

### 通用规范

- 使用中文名称，清晰明确
- 使用连字符（-）分隔单词
- 版本号格式：v1.0、v1.1等
- 日期格式：YYYY-MM-DD

### 示例

```
✅ 好的命名：
- OpenCode安装指南.md
- API设计文档-v1.0.yaml
- ReAct编排示例-20240303.json

❌ 不好的命名：
- doc1.md
- 新建文档.txt
- api设计.yaml
```

## 🔐 文档安全

### 敏感信息处理

**需要保密的信息**：
- 灵知平台账号密码
- API Key和密钥
- OpenCode/Trae账号信息
- 国网四川内部文档

**处理方式**：
- 账号密码：单独存储，不提交到Git
- API Key：使用环境变量，不硬编码
- 内部文档：脱敏后上传到知识库

### Git提交规范

**可以提交**：
- 公开的文档和指南
- 示例代码（不含密钥）
- Mock服务代码

**不可以提交**：
- 账号密码清单
- API密钥
- 脱敏前的原始文档
- 其他敏感信息
