## 1. 基础配置

### 1.1 创建准备（5步骤）

1. **评估当前需求**：定义业务场景的目的和目标
2. **评估AI能力**：匹配需求与平台能力（智能对话、知识库搜索、信息分类、信息提取等）
3. **画布编排**：拖拽模块/智能体/工具/MCP，连接数据节点
4. **用户交互设计**：设计对话引导、异常处理、用户体验
5. **测试与实验**：持续测试调优

### 1.2 智能体配置（3部分）

- **基本信息配置**：头像、名称、开场白等
- **规划配置**：模块化编排实现功能
- **分发配置**：删除、保存、发布等操作

### 1.3 配置页面详细项

| 配置项 | 说明 |
|--------|------|
| 智能体头像 | 可自定义上传、设定文字头像或选择平台已有头像 |
| 智能体名称 | 为当前智能体命名，使用者会第一时间查看此名称 |
| 智能体描述 | 简单描述该智能体的功能/场景/其他信息 |
| 对话背景 | 默认为白色，可选择精美聊天背景或自定义上传 |
| 开场白 | 对话开始前，使用者首先看到智能体发送的消息 |
| 问题建议 | 预设问题引导，最多可添加6个，在首次对话时呈现 |
| 对话模型 | 设定默认「智能对话」模块所使用的模型 |
| 提示词 | 设定默认「知识库搜索」模块所关联的知识库 |
| 知识库 | 对话过程中将对知识库进行内容检索，单个智能体可选择多个知识库 |
| 是否显示知识库引用 | 通过知识库搜索回复的答案，是否展示参考信息的文档来源 |

## 2. 规划概述

### 2.1 界面功能

#### 模块区（左侧）
平台模块分为4类：
- **系统模块**：链接AI能力，对输入文本进行处理（信息加工、知识库搜索等）
- **智能体**：调用已上线的智能体，进行更复杂能力规划
- **工具**：常用工具打包，扩充第三方能力，支持自定义开发
- **MCP**：通用协议打通Agent与外部工具、数据源的交互

#### 画布区（中央）
- 拖拽组件移入画布，通过连线组合模块能力
- 左侧接口为输入，右侧接口为输出
- 只能将"输入和输出"连接，不能连接"输入和输入"或"输出和输出"
- 连接错误可点击线中间的"x"删除

#### 功能区（上方）
- **画布功能区**：导入/导出编排、试运行调试
- **智能体功能区**：分享（Http应用、API服务、MCP Server、应用集成）、删除、保存、发布

### 2.2 节点类型与连接原则

| 节点颜色 | 数据类型 | 说明 |
|----------|----------|------|
| 黄色 | 布尔型 | 与其他布尔型节点连接 |
| 蓝色 | 字符串 | 文本、图片、文档信息 |
| 紫色 | 知识库搜索结果 | 仅用于输入输出知识库搜索结果 |
| 红色 | 任意类型 | 仅在For Each模块中，需为JSON数组或对象 |

**连接原则**：同类型节点互相连接，不同类型节点不可连接

## 3. 模块结构

每个模块由三部分组成：
1. **左上角 - 模块输入区**：连接节点将信息输入
2. **中间部分 - 模块设置区**：设置模块参数、提示词等
3. **右下角 - 模块输出区**：将模块执行后的信息输出

## 4. 基础模块详解

### 4.1 用户提问（入口模块）

**功能**：获取用户提问/选择内容，并传输给其他模块

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| 联动激活 | 布尔型 | 上游所有条件均为True时激活 |
| 任一激活 | 布尔型 | 上游任一条件为True时激活 |
| 输入文本 | 字符串 | 允许用户通过输入框输入文本（默认打开） |
| 上传文档 | 字符串 | 允许用户上传文档（最多9个，每个50MB，支持pdf/doc/docx/txt） |
| 上传图片 | 字符串 | 允许用户上传图片（最多9个，单个30M，支持png/jpg/jpeg） |
| 文档审查 | 字符串 | 开启文档审查功能，配合「文档审查」模块使用 |
| 添加按钮 | - | 可作为明确执行节点或上游反馈 |
| 未点击按钮 | 布尔型 | 用户未点击按钮时值为true |
| 按钮节点 | 布尔型 | 点击按钮且对话框有输入则执行 |
| 模块运行结束 | 布尔型 | 模块运行结束输出True，触发下游流程 |

**输出**：文本信息、文档信息、图片信息、未点击按钮、模块运行结束

---

### 4.2 智能对话

**功能**：借助AI能力，将用户发送内容通过大语言模型处理并回复

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| 联动激活/任一激活 | 布尔型 | 激活条件 |
| 信息输入 | 字符串 | 连接上游输出的文本 |
| 图片输入 | 字符串 | 连接上游输出的图片 |
| 知识库搜索结果 | 知识库类型 | 连接知识库搜索组件结果 |
| 聊天上下文 | - | 可设置0-6条聊天记录作为输入 |
| 选择模型 | - | 选择大语言模型 |
| 系统提示词 | - | 用prompt设定模型角色和行为模式 |
| 用户提示词 | - | 用prompt说明需要处理的任务 |
| 回复对用户可见 | - | 关闭后执行但不展示回复（用于中间流程） |
| 回复创意性 | - | 0-1，越小越严谨，越大越发散 |
| 核采样TOP_P | - | 0-1，控制输出多样性 |
| 回复字数上限 | - | 控制回复字数（100-8192） |

**输出**：回复结束（是/否）、回复内容、模块运行结束

### 4.5 知识库搜索

**功能**：在知识库中搜索相关问题与解答，以自然语言输出

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| 联动激活/任一激活 | 布尔型 | 激活条件 |
| 信息输入 | 字符串 | 连接上游输出的文本 |
| 关联的知识库 | - | 选择搜索的知识库（最多5个） |
| 相似度阈值 | - | 0-1，低于此阈值的切片将被过滤 |
| 向量相似度权重 | - | 0-1，默认1（只使用向量检索） |
| 召回数 | - | 0-100，召回的相关切片数量 |
| 扩展上下文 | - | 开启后召回命中分段上下各1个分段 |
| 权限访问控制 | - | 开启后仅检索当前用户有权限的知识库 |
| 是否开启重排序 | - | 重排序提高精确度但降低速度 |
| 重排序模型 | - | 可选择bce-rerank模型 |
| 重排序召回数 | - | 0-20，重排序后召回数量 |

**输出**：
- 未搜索到相关知识（布尔型）
- 搜索到相关知识（布尔型）
- 模块运行结束（布尔型）
- 知识库搜索结果（数组格式）

## 6. MCP

### 6.1 创建MCP

**步骤**：
1. 进入MCP管理页面
2. 点击"创建MCP服务"
3. 填写服务信息：
   - 服务名称
   - 描述
   - 安装方式（目前只支持SSE）
   - MCP配置（JSON格式）

**配置示例**：
```json
{
  "mcpServers": {
    "ocr_edu": {
      "url": "https://aip.baidubce.com/mcp/ocr_general/sse?Authorization=Bearer+xxx"
    }
  }
}
```

---

### 6.2 MCP调用

**步骤**：
1. 进入智能体-规划页面，左侧MCP展示已接入服务
2. 拖拽MCP服务到画布
3. 选择MCP所用的工具
4. 通过连线把MCP加入到工作流中

---

## 7. 知识库

### 7.1 创建知识库

**步骤**：
1. 点击左侧"知识库"选项卡
2. 点击"创建知识库"
3. 填写基本信息：
   - 知识库名称（必填）
   - 描述（选填）
4. 高级设置：
   - 解析方法（法律、论文、图书、手册、FAQ等）
   - 嵌入模型（一旦选择不可更改）
   - 块token数（平衡精度和效率）
5. 点击"确定"完成创建

---

### 7.2 上传文件

**步骤**：
1. 进入知识库的"数据集"页面
2. 点击"上传"选择文件
3. 上传完成后点击"更新到知识库"
4. 点击"解析"按钮
5. 系统自动切片并嵌入数据
6. 状态更新为"成功"后可被检索

### 7.3 检索测试

文件解析完成后，可在检索测试页面测试查询效果，调试参数：

| 参数 | 说明 |
|------|------|
| 相似度阈值 | 低于此阈值的切片将被过滤 |
| 向量相似度权重 | 默认1（只使用向量检索） |
| 召回数 | 召回的相关切片数量 |
| 是否开启重排序 | 提高精确度但降低速度 |
| 重排序召回数 | 重排序后召回数量 |

### 9.2 系统内置函数

**语法**：`[@["functionName","args1","args2",...]@]` 表示函数调用

#### 日期处理
- `DateFormatFromMills` / `DateFormatFromSeconds`：日期格式化
- `DateStrToMills` / `DateStrToSeconds`：日期转换为毫秒/秒
- `GetDateFromNow`：日期加减

#### 字符串处理
- `ExtractAroundStr`：提取包围的子字符串
- `ExtractStrByRegex`：提取第一个符合正则的字符串
- `ExtractArrByRegex`：提取所有符合正则的字符串
- `Replace` / `ReplaceRegex`：字符串替换/正则替换
- `EscapeStr`：字符串转义
- `Concat`：字符串拼接
- `ConcatArray`：数组拼接

#### JSON处理
- `ExtractJsonObj`：提取json对象
- `ExtractJsonArr`：提取json数组
- `ExtractJsonByRegex`：正则提取json

#### 其他
- `UrlEncode` / `UrlDecode`：URL编码/解码
- `AutoSpecifyResponseLanguage`：自动检测指定回复语言
- `RSAEncode`：RSA加密
- `CaseWhenThen`：根据条件赋值
- `EqualsAny`：是否与任意候选值相等
- `DateStrReFormat`：日期格式转换
- `JionlpParseTime`：自然语言转日期
