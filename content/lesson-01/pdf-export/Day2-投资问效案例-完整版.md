## 第一部分：案例概述与业务理解

### 1.1 案例背景

<!-- 
【设计说明】
本案例选取国网四川省电力公司真实业务场景，具有以下特点：
1. 业务逻辑清晰（8项指标评分）
2. 数据来源明确（3个API接口）
3. 输出结果标准化（评估报告）
4. 适合教学（覆盖Agent开发核心技能）
-->

**业务痛点**:
```
┌─────────────────────────────────────────────────────────────┐
│                     传统投资问效的痛点                        │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  📊 数据维度多                                                │
│     • 项目基本信息（10+字段）                                  │
│     • 投资数据（概算/结算/偏差）                               │
│     • 运行指标（容载比/N-1/负载率）                            │
│     • 可研预测数据                                            │
│                                                               │
│  🔄 比对工作繁重                                              │
│     • 预测值 vs 实际值对比                                     │
│     • 多维度数据交叉验证                                       │
│     • 人工计算容易出错                                         │
│                                                               │
│  ⏱️  效率低下                                                 │
│     • 单个项目分析需2-3小时                                    │
│     • 年度评估项目数量大（115个）                              │
│     • 人工难以全覆盖                                          │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

**智能解决方案**:
```
输入: 可研报告(docx/pdf) 或 项目检索号
        ↓
处理: Agent自动解析 → 提取检索号 → 调用API获取数据 → 评分计算
        ↓
输出: 结构化投资问效分析报告
```

### 1.2 核心业务流程

<!-- 
【流程说明】
这是本案例的核心数据流，理解此流程至关重要：
1. 检索号是核心标识符，串联所有数据
2. 3个API可以并行调用，提高效率
3. 评分计算依赖所有数据就绪
-->

```
关键依赖链：

用户输入（可研报告/检索号）
        │
        ▼
文档解析（提取retrieval_no）
        │
        ▼
        ├─→ 基本信息API ──→ 电压等级、项目类型、日期
        ├─→ 投资数据API ──→ 概算、结算金额
        └─→ 运行指标API ──→ 容载比、负载率、N-1通过率
                │
                ▼
        数据整合与比对
                │
        ├─→ 8项指标评分
        └─→ 差异分析
                │
        生成评估报告

⚠️ 注意：检索号是串联所有数据的核心标识符！
```

### 1.3 8项评估指标

<!-- 
【评分体系说明】
这是业务规则的核心，Agent需要根据这些规则进行评分。
每个指标都有明确的权重和评分逻辑。
-->

| 指标类别 | 具体指标 | 权重 | 数据来源 | 评分逻辑 |
|---------|---------|------|---------|---------|
| **电网运行** | 容载比 | 15% | 运行指标API | 3年都在标准区间=满分 |
| **电网运行** | N-1通过率改善 | 15% | 运行指标API | 改善幅度≥10%=满分 |
| **设备状态** | 设备负载率 | 10% | 运行指标API | 达到预期=满分 |
| **负荷预测** | 负荷偏差率 | 15% | 可研报告+运行指标 | 偏差≤10%=满分 |
| **设备状态** | 最大负载率 | 10% | 运行指标API | 无重过载=满分 |
| **设备状态** | 平均负载率 | 10% | 运行指标API | 利用效率高=满分 |
| **投资控制** | 投资结余率 | 15% | 投资数据API | 结余5-15%=满分 |
| **项目管理** | 建设时长 | 10% | 基本信息API | 按期完成=满分 |

**评分示例**（容载比）：
```
110kV项目容载比标准：[1.8, 2.2]

第1年实际值：2.0 → ✓ 在区间内
第2年实际值：2.3 → ✗ 超出上限
第3年实际值：1.9 → ✓ 在区间内

评分：2年在区间内 → 11分（满分15分）
评价：容载比基本合理，第2年偏高需关注
```

**代码块**: JSON解析
```yaml
# 【目的】将JSON字符串转为结构化数据，便于后续引用

节点名称: JSON解析
语言: JavaScript

入参:
  - json_str: {{project_info_json}}  # 上游模块的输出

代码: |
  /**
   * 【功能】解析JSON字符串，提取关键字段
   * 【输入】json_str - JSON格式的字符串
   * 【输出】结构化的项目信息
   */
  function userFunction(param) {
      try {
          // 【步骤1】解析JSON字符串
          var data = JSON.parse(param['json_str']);
          
          // 【步骤2】提取并格式化字段
          var result = {
              // 【核心】检索号是后续所有API调用的关键
              retrieval_no: data.retrieval_no || '',
              project_name: data.project_name || '',
              voltage_level: parseInt(data.voltage_level) || 0,
              project_type: data.project_type || '',
              forecast_load: parseFloat(data.forecast_load) || 0,
              construction_goal: data.construction_goal || '',
              start_date: data.start_date || '',
              completion_date: data.completion_date || '',
              
              // 【状态】标记解析是否成功
              success: true,
              parse_time: new Date().toISOString()
          };
          
          // 【验证】确保检索号存在（核心字段）
          if (!result.retrieval_no) {
              return {
                  success: false,
                  error: '未找到检索号，无法继续查询',
                  error_code: 'MISSING_RETRIEVAL_NO'
              };
          }
          
          return result;
          
      } catch(e) {
          // 【错误处理】JSON解析失败
          return {
              success: false,
              error: 'JSON解析失败: ' + e.message,
              error_code: 'JSON_PARSE_ERROR',
              raw_input: param['json_str'].substring(0, 100) // 记录前100字符用于调试
          };
      }
  }

出参:
  - retrieval_no (字符串)  # 【核心】用于API调用
  - project_name (字符串)
  - voltage_level (数字)
  - project_type (字符串)
  - forecast_load (数字)
  - success (布尔型)       # 用于分支判断
  - error (字符串)         # 错误信息
```

#### Step 5: HTTP模块配置（30分钟）

**模块**: HTTP调用 × 3（并行）

```yaml
# ═══════════════════════════════════════════════════════════════
# 【API 1】获取项目基本信息
# ═══════════════════════════════════════════════════════════════
节点名称: 获取基本信息

# 【参数】从上游模块获取retrieval_no
请求参数:
  - 参数名: retrieval_no
  - 参数值: {{JSON解析.retrieval_no}}  # 【引用】上游模块的输出

请求头:
  Content-Type: application/json

# 【响应处理】
成功响应码: 200
输出变量: basic_info

# 【错误处理】
# 非200响应码时，走错误分支
错误分支提示: |
  查询项目基本信息失败。
  
  可能原因：
  1. 检索号 {{JSON解析.retrieval_no}} 不存在
  2. 数据服务暂时不可用
  
  建议：检查检索号是否正确，稍后重试。
```

#### Step 6: 数据整合模块（20分钟）

<!-- 
【设计说明】
数据整合是业务逻辑的核心，需要：
1. 解析3个API的返回数据
2. 计算派生指标（偏差率等）
3. 统一数据格式
4. 为后续报告生成准备数据
-->

**模块**: 代码块（数据整合）

```yaml
节点名称: 整合项目数据

入参:
  - basic_info: {{获取基本信息.output}}      # API 1返回
  - investment_info: {{获取投资数据.output}} # API 2返回
  - operation_info: {{获取运行指标.output}}  # API 3返回
  - project_info: {{JSON解析.output}}        # 文档解析结果

语言: JavaScript

代码: |
  /**
   * 【功能】整合3个API的数据，计算派生指标
   * 【输入】3个API的原始返回 + 文档解析信息
   * 【输出】统一格式的项目数据
   */
  function userFunction(param) {
      try {
          // 【步骤1】解析API返回的JSON
          var basic = JSON.parse(param['basic_info']);
          var investment = JSON.parse(param['investment_info']);
          var operation = JSON.parse(param['operation_info']);
          var project = JSON.parse(param['project_info']);
          
          // 【步骤2】提取基本信息
          var basicData = basic.data || {};
          var investData = investment.data || {};
          var operData = operation.data || {};
          
          // 【步骤3】计算投资偏差
          var estimated = investData['初设概算项目总投资（万元）'] || 0;
          var actual = investData['结算批复项目动态投资（万元）'] || 0;
          var costVariance = estimated > 0 ? 
              ((actual - estimated) / estimated * 100).toFixed(2) : 0;
          
          // 【步骤4】计算负荷偏差
          var forecast = project.forecast_load || 0;
          var actualLoad = operData['投运后1年（新增设备）实际负荷值(万千瓦)'] || 0;
          var loadVariance = forecast > 0 ? 
              Math.abs((forecast - actualLoad) / forecast * 100).toFixed(2) : 0;
          
          // 【步骤5】构建统一数据对象
          var result = {
              // 【基本信息】
              retrieval_no: basicData['检索号'],
              project_name: basicData['项目名称'],
              voltage_level: basicData['电压等级'],
              project_type: basicData['项目类型'],
              location: basicData['所在区县'],
              dispatch_name: basicData['调度命名'],
              start_date: basicData['实际开工日期'],
              completion_date: basicData['实际投产日期'],
              
              // 【投资数据】
              estimated_cost: estimated,
              actual_cost: actual,
              cost_variance: parseFloat(costVariance),
              
              // 【运行指标】
              load_year1: actualLoad,
              load_year2: operData['投运后2年（新增设备）实际负荷值(万千瓦)'],
              load_year3: operData['投运后3年（新增设备）实际负荷值(万千瓦)'],
              max_load_rate: operData['投运后1年最大负载率'],
              avg_load_rate: operData['投运后1年平均负载率'],
              device_load_rates: operData['项目投运次年关联设备负载率'],
              n1_rate_before: operData['投运前一年本项目同电压等级所在地市N-1通过率'],
              n1_rate_after: operData['投运当年年项目电压等级所在地市N-1通过率'],
              
              // 【可研预测】
              forecast_load: forecast,
              load_variance: parseFloat(loadVariance),
              
              // 【元信息】
              data_quality: {
                  has_basic: !!basicData['检索号'],
                  has_investment: estimated > 0,
                  has_operation: actualLoad > 0,
                  completeness: 'full'  // full/partial/none
              },
              
              // 【格式化输出】便于调试
              formatted: JSON.stringify({
                  basic: basicData,
                  investment: investData,
                  operation: operData
              }, null, 2)
          };
          
          return {
              success: true,
              data: result,
              json: JSON.stringify(result, null, 2)
          };
          
      } catch(e) {
          return {
              success: false,
              error: '数据整合失败: ' + e.message,
              stack: e.stack
          };
      }
  }

出参:
  - success (布尔型)
  - data (对象)        # 结构化数据
  - json (字符串)      # JSON格式字符串，便于在提示词中引用
  - error (字符串)
```

## 第五部分：常见问题

### Q1: 文档解析失败怎么办？

**可能原因**:
- 文件格式不支持（仅支持.docx/.pdf/.doc）
- 文件被加密或损坏
- 文档内容为空

**解决方案**:
1. 检查文件格式
2. 尝试用其他软件打开验证
3. 重新保存为标准格式

### Q2: HTTP调用返回400错误？

**可能原因**:
- 参数名错误（大小写敏感）
- 缺少必需参数
- 参数值格式错误

**解决方案**:
1. 检查参数名是否与API文档一致
2. 确认retrieval_no已正确提取
3. 查看API文档中的示例

### Q3: 报告生成质量不高？

**优化方法**:
1. 优化系统提示词，提供更多示例
2. 调整模型温度参数
3. 在提示词中指定输出格式
