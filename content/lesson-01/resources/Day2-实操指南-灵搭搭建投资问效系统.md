# Day2 实操指南：灵搭平台搭建投资问效系统

## 课程目标

通过本实操指南，你将学会：
1. 在灵搭平台上搭建多智能体协作流程
2. 配置HTTP模块调用外部API
3. 使用信息提取模块处理非结构化数据
4. 实现基本的报告生成Agent

**预计用时**：4课时（160分钟）

---

## 第一部分：环境准备与基础配置（20分钟）

### 1.1 登录灵搭平台

**操作步骤**：
1. 打开灵搭平台网址（内网地址）
2. 使用统一身份认证登录
3. 进入"智能体开发"模块

### 1.2 创建主智能体

**操作路径**：智能体开发 → 新建Agent

**基础配置**：
```yaml
Agent名称: 投资问效分析助手
描述: 基于可研报告和运行数据，自动生成投资问效分析报告
头像: 选择"数据分析"相关图标

开场白: |
  您好！我是投资问效分析助手。
  我可以帮您：
  1. 分析35kV及以上输变电工程可研报告
  2. 获取项目投资、运行数据
  3. 生成投资问效评估报告
  
  请上传可研报告文件（支持.docx/.pdf格式）

问题建议:
  - "分析成都丛树110kV输变电工程"
  - "上传可研报告并生成评估"
  - "查看投资问效评分标准"
```

### 1.3 配置大模型参数

**参数设置**：
| 参数 | 值 | 说明 |
|------|-----|------|
| 模型 | 光明电力大模型V2 | 电力领域专用模型 |
| 回复创意性 | 0.1 | 保持输出稳定 |
| 回复字数上限 | 8000 | 报告需要较长输出 |
| 上下文条数 | 10 | 保持对话连贯 |

### 2.2 步骤详解

#### Step 1: 用户输入与文档接收（10分钟）

**拖入模块**：
1. **用户提问**（左侧系统模块）
2. **信息分类**（判断用户意图）

**模块配置**：

**用户提问模块**：
```
节点名称: 用户输入
保存用户输入: 是
存储变量名: user_input
```

**信息分类模块**（连接用户提问）：
```
节点名称: 意图识别
系统提示词: |
  你是投资问效系统的意图识别助手。
  请判断用户的输入属于哪种类型：
  
  可选类型：
  1. 上传文档 - 用户上传了文件或提到"分析""评估"
  2. 直接查询 - 用户提供了项目检索号或项目名称
  3. 其他咨询 - 询问系统功能、评分标准等
  
  只输出类型名称（上传文档/直接查询/其他咨询），不要其他内容。

分类结果: intent_type
```

**连接方式**：
- 用户提问 → 信息分类
- 信息分类输出（字符串）→ 后续分支

#### Step 2: 文档解析与信息提取（20分钟）

**拖入模块**：
1. **知识库搜索**（文档解析）
2. **信息提取**

**知识库搜索模块配置**：
```
节点名称: 文档解析
选择知识库: [创建新的知识库]
  知识库名称: 可研报告解析库
  文档上传: 支持.docx/.pdf
  解析方式: 完整文本解析
  
相似度阈值: 0.5
召回数量: 1
搜索类型: 全文检索

输出: knowledge_result
```

**信息提取模块配置**：
```
节点名称: 提取关键信息
系统提示词: |
  从可研报告文本中提取以下关键信息：
  
  提取字段：
  1. retrieval_no: 检索号（格式如：798-K131538-A01）
  2. project_name: 项目名称
  3. voltage_level: 电压等级（35/110/220等数字）
  4. project_type: 项目类型（经济效益类/安全效益类/社会效益类）
  5. forecast_load: 预测负荷值（数字，单位MW）
  6. construction_goal: 建设目标
  7. start_date: 开工日期（ISO格式）
  8. completion_date: 投产日期（ISO格式）
  
  以JSON格式输出，不要其他内容：
  {
    "retrieval_no": "",
    "project_name": "",
    "voltage_level": 0,
    "project_type": "",
    "forecast_load": 0,
    "construction_goal": "",
    "start_date": "",
    "completion_date": ""
  }

提取字段: project_info_json
```

**小技巧**：使用代码块模块将JSON字符串转为结构化数据：
```javascript
// 代码块模块配置
入参: json_str (字符串)
代码:
function userFunction(param) {
    try {
        var data = JSON.parse(param['json_str']);
        return {
            retrieval_no: data.retrieval_no || '',
            project_name: data.project_name || '',
            voltage_level: parseInt(data.voltage_level) || 0,
            project_type: data.project_type || '',
            forecast_load: parseFloat(data.forecast_load) || 0,
            success: true
        };
    } catch(e) {
        return { success: false, error: e.message };
    }
}
```

#### Step 3: HTTP模块配置 - 调用业务API（30分钟）

**拖入模块**：HTTP调用（需要3个，分别调用3个API）

**API 1: 获取项目基本信息**

```
节点名称: 获取基本信息
请求方法: GET
请求URL: http://127.0.0.1:5000/api/project/basic
请求参数:
  - 参数名: retrieval_no
  - 参数值: {{project_info.retrieval_no}}

请求头: 
  Content-Type: application/json

响应处理:
  成功响应码: 200
  输出变量: basic_info
```

**API 2: 获取项目投资数据**

```
节点名称: 获取投资数据
请求方法: GET
请求URL: http://127.0.0.1:5000/api/project/investment
请求参数:
  - 参数名: retrieval_no
  - 参数值: {{project_info.retrieval_no}}

响应处理:
  成功响应码: 200
  输出变量: investment_info
```

**API 3: 获取项目运行指标**

```
节点名称: 获取运行指标
请求方法: GET
请求URL: http://127.0.0.1:5000/api/project/operation
请求参数:
  - 参数名: retrieval_no
  - 参数值: {{project_info.retrieval_no}}

响应处理:
  成功响应码: 200
  输出变量: operation_info
```

**并行调用配置**：
```
使用"任一激活"模式：

        ┌─→ [获取基本信息]
        │
[提取] ─┼─→ [获取投资数据] ──→ [数据整合]
        │
        └─→ [获取运行指标]

注意：3个HTTP模块使用"任一激活"连接到"数据整合"代码块
```

#### Step 4: 数据整合与处理（20分钟）

**拖入模块**：代码块（数据整合）

**代码块配置**：
```
节点名称: 整合项目数据
入参:
  - basic_info (字符串)
  - investment_info (字符串)
  - operation_info (字符串)
  - project_info (字符串)

语言: JavaScript

代码:
function userFunction(param) {
    try {
        var basic = JSON.parse(param['basic_info']);
        var investment = JSON.parse(param['investment_info']);
        var operation = JSON.parse(param['operation_info']);
        var project = JSON.parse(param['project_info']);
        
        // 整合数据
        var result = {
            // 基本信息
            retrieval_no: basic.data['检索号'],
            project_name: basic.data['项目名称'],
            voltage_level: basic.data['电压等级'],
            project_type: basic.data['项目类型'],
            location: basic.data['所在区县'],
            
            // 投资数据
            estimated_cost: investment.data['初设概算项目总投资（万元）'],
            actual_cost: investment.data['结算批复项目动态投资（万元）'],
            
            // 运行指标
            load_year1: operation.data['投运后1年（新增设备）实际负荷值(万千瓦)'],
            max_load_rate: operation.data['投运后1年最大负载率'],
            avg_load_rate: operation.data['投运后1年平均负载率'],
            device_load_rates: operation.data['项目投运次年关联设备负载率'],
            
            // 可研预测
            forecast_load: project.forecast_load,
            
            // 计算字段
            cost_variance: ((investment.data['结算批复项目动态投资（万元）'] - 
                           investment.data['初设概算项目总投资（万元）']) / 
                           investment.data['初设概算项目总投资（万元）'] * 100).toFixed(2),
            
            load_variance: project.forecast_load > 0 ? 
                          ((project.forecast_load - operation.data['投运后1年（新增设备）实际负荷值(万千瓦)']) / 
                           project.forecast_load * 100).toFixed(2) : 0
        };
        
        return {
            success: true,
            data: result,
            formatted: JSON.stringify(result, null, 2)
        };
    } catch(e) {
        return {
            success: false,
            error: e.message
        };
    }
}

出参:
  - success (布尔型)
  - data (字符串)
  - formatted (字符串)
  - error (字符串)
```

#### Step 5: 报告生成（20分钟）

**拖入模块**：智能对话（生成报告）

**智能对话模块配置**：
```
节点名称: 生成分析报告
系统提示词: |
  你是电力投资问效分析专家。根据提供的项目数据，生成专业的投资问效分析报告。
  
  报告要求：
  1. 包含项目基本信息、投资情况、运行指标
  2. 计算关键差异（投资偏差、负荷偏差等）
  3. 给出客观评价和建议
  4. 使用Markdown格式
  5. 语言专业、简洁

用户问题: |
  请根据以下项目数据生成投资问效分析报告：
  
  {{integrated_data.formatted}}

模型: 光明电力大模型V2
回复字数上限: 4000
```

**连接流程**：
```
[整合项目数据] ──→ [生成分析报告] ──→ [用户展示]
```

### 2.3 异常处理分支（10分钟）

**添加错误处理流程**：

**分支1: API调用失败处理**
```
HTTP模块 ──→ [判断响应码]
              │
              ├─→ 200 ──→ 正常处理
              │
              └─→ 非200 ──→ [智能对话-错误提示] ──→ [确定回复]

错误提示模板:
"抱歉，查询项目数据时出现错误（错误码：{{status_code}}）。
可能原因：
1. 检索号不正确
2. 项目数据尚未录入
3. 数据服务暂时不可用

建议：
- 请检查检索号是否正确
- 稍后重试或联系管理员"
```

**分支2: 数据缺失处理**
```
[代码块] ──→ [判断success]
              │
              ├─→ true ──→ 继续
              │
              └─→ false ──→ [智能对话-数据处理错误] ──→ [确定回复]
```

### 3.3 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| HTTP调用返回400 | 参数缺失或格式错误 | 检查参数名是否正确 |
| JSON解析失败 | 返回格式非JSON | 检查API响应格式 |
| 信息提取不完整 | 提示词不够精确 | 优化提取提示词 |
| 数据整合错误 | 字段名不匹配 | 统一字段命名规范 |
| 流程不执行 | 连接条件未满足 | 检查激活条件 |

## 第四部分：优化与完善（课后作业）

### 4.1 可优化方向

1. **缓存机制**
   - 已查询的项目数据缓存到知识库
   - 避免重复调用API

2. **评分细化**
   - 实现8项指标的独立评分Agent
   - 使用代码块计算具体分数

3. **可视化增强**
   - 生成数据对比图表
   - 使用信息加工模块格式化输出

4. **多轮对话**
   - 支持用户追问细节
   - 保持上下文连贯

### 4.2 扩展功能建议

1. **批量处理**：使用循环模块处理多个项目
2. **历史记录**：保存分析历史到数据库
3. **报告导出**：支持导出PDF/Word格式
