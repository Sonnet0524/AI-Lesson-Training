# 技术栈速查

## 核心技术

### Agentic AI

**定义**：具有自主决策和执行能力的 AI 系统

**核心特征**：
- 🎯 目标驱动
- 🔄 自主迭代
- 🛠️ 工具调用
- 💭 推理规划

**应用场景**：智能问数、智能写作、工单处理

---

### ReAct 模式

**定义**：推理+ 行动的循环模式

**核心循环**：
```
Thought → Action → Observation → Thought
```

**关键要素**：
- Thought：分析当前状态，规划下一步
- Action：执行具体操作（调用工具）
- Observation：观察执行结果
- 循环：根据结果继续思考

---

## 开发平台

### 灵知平台

**定位**：低代码 Agent 开发平台

**核心模块**：
- 用户提问：输入入口
- 智能对话：LLM 交互
- 知识库搜索：RAG 检索
- 信息分类：文本分类
- 信息提取：结构化提取

**参数建议**：
| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 相似度阈值 | 0.5-0.7 | 知识库召回精度 |
| 召回数 | 3-10 | 检索结果数量 |
| 聊天上下文 | 2-5 条 | 历史对话轮数 |
| 回复创意性 | 0-0.8 | 输出随机性 |

## 知识库技术

### RAG（检索增强生成）

**原理**：检索 + 生成

**流程**：
1. 用户提问
2. 向量化查询
3. 相似度检索
4. 上下文增强
5. LLM 生成答案

**关键参数**：
- 文档分块大小：500-1000 字
- 重叠长度：100-200 字
- 嵌入模型：建议使用平台默认

## 常见问题

### Q1: 如何选择 LLM 模型？

**推荐**：
- 中文场景：通义千问、文心一言
- 英文场景：GPT-4、Claude
- 平衡选择：灵知平台默认模型

---

### Q2: OpenCode 和 Trae 如何选择？

**建议**：
- 预算有限：OpenCode（免费）
- 需要多模态：Trae（付费功能更强）
- 培训场景：OpenCode（推荐）

---

### Q3: 知识库文档如何准备？

**要求**：
- 格式：PDF、Word、TXT
- 语言：中文
- 内容：脱敏处理
- 大小：单文件 < 10MB

---

### Q4: Agent 开发流程是什么？

**步骤**：
1. 需求分析
2. 画布设计
3. 模块配置
4. 测试优化
5. API 发布
