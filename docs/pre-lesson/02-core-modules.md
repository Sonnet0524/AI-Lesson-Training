# Pre-Lesson 第2次直播：核心模块与编排

## 基本信息

**时间**：Day 1 前2天

**总时长**：90分钟（授课60分钟 + 交流30分钟）

**学习目标**：
- 掌握5个核心模块的功能和参数
- 理解常见编排模式

## 课程内容

### Part 1：核心模块详解（30分钟）

#### 模块1：用户提问模块（5分钟）

**功能**：获取用户提问/选择内容，传输给其他模块

**界面展示**：
```
┌─────────────────────────────┐
│ 用户提问                     │
├─────────────────────────────┤
│ [模块名称] 用户提问          │
│                             │
│ □ 输入文本（默认打开）       │
│ □ 上传文档（最多9个，50MB）  │
│ □ 上传图片（最多9个，30MB）  │
│ □ 文档审查                  │
│ □ 添加按钮                  │
├─────────────────────────────┤
│ 输出：                       │
│ ○ 文本信息（蓝色）          │
│ ○ 文档信息（蓝色）          │
│ ○ 图片信息（蓝色）          │
│ ○ 模块运行结束（黄色）      │
└─────────────────────────────┘
```

**核心参数**：
| 参数 | 说明 |
|------|------|
| 输入文本 | 是否显示输入框 |
| 上传文档 | 支持pdf/doc/docx/txt，最多9个 |
| 上传图片 | 支持jpg/png，最多9个 |
| 添加按钮 | 添加可点击按钮，用于明确执行 |

**使用场景**：
- 所有Agent的入口模块
- 需要用户上传文件的场景
- 需要按钮交互的场景

#### 模块2：智能对话模块（8分钟）

**功能**：通过大语言模型处理并回复用户

**界面展示**：
```
┌─────────────────────────────┐
│ 智能对话                     │
├─────────────────────────────┤
│ [模块名称] 智能对话          │
│                             │
│ [模型选择] GPT-4 / 文心...   │
│                             │
│ 系统提示词：                 │
│ ┌─────────────────────────┐ │
│ │ 你是一个智能助手...      │ │
│ └─────────────────────────┘ │
│                             │
│ 聊天上下文：[0-6]           │
│ 回复创意性：[0-1]           │
│ 回复字数上限：[100-8192]    │
│ 回复对用户可见：□           │
├─────────────────────────────┤
│ 输出：                       │
│ ○ 回复结束（黄色）          │
│ ○ 回复内容（蓝色）          │
│ ○ 模块运行结束（黄色）      │
└─────────────────────────────┘
```

**核心参数详解**：

| 参数 | 范围 | 建议 | 说明 |
|------|------|------|------|
| 聊天上下文 | 0-6 | 3-5 | 保留多少轮历史对话 |
| 回复创意性 | 0-1 | 知识库场景0，创作场景0.6-0.8 | 0最严谨，1最发散 |
| 核采样TOP_P | 0-1 | 与创意性二选一 | 控制生成多样性 |
| 回复字数上限 | 100-8192 | 根据场景 | 限制回复长度 |
| 回复对用户可见 | 是/否 | 中间流程可设为否 | 是否显示给用户 |

**系统提示词模板**：
```
【角色定位】
你是一个[角色描述]，专门负责[任务描述]。

【回答要求】
1. [要求1]
2. [要求2]
3. [要求3]

【注意事项】
- [注意点1]
- [注意点2]
```

**案例**：
```
场景：电力知识问答助手

系统提示词：
你是国网四川电力的智能客服助手，专门回答电力业务相关问题。

回答要求：
1. 准确引用知识库内容，不要编造信息
2. 语言简洁明了，避免专业术语堆砌
3. 如遇不确定问题，引导用户咨询人工客服

聊天上下文：3
回复创意性：0
```

#### 模块3：信息分类模块（5分钟）

**功能**：将用户问题分类，针对不同类型执行不同操作

**界面展示**：
```
┌─────────────────────────────┐
│ 信息分类                     │
├─────────────────────────────┤
│ [模块名称] 信息分类          │
│                             │
│ 提示词：                     │
│ ┌─────────────────────────┐ │
│ │ 请判断用户问题属于哪种类型│ │
│ └─────────────────────────┘ │
│                             │
│ 标签设置：                   │
│ + 添加标签                   │
│ ┌─────────────────────────┐ │
│ │ 标签1: 咨询              │ │
│ │ 标签2: 投诉              │ │
│ │ 标签3: 建议              │ │
│ │ 标签4: 其他              │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ 输出：                       │
│ ○ 咨询（黄色）              │
│ ○ 投诉（黄色）              │
│ ○ 建议（黄色）              │
│ ○ 其他（黄色）              │
│ ○ 分类结果（蓝色）          │
│ ○ 模块运行结束（黄色）      │
└─────────────────────────────┘
```

**使用场景**：
- 工单分类：故障报修/业务咨询/投诉建议
- 问题类型识别：技术问题/业务问题/其他
- 多流程分支：根据分类走不同处理流程

**案例**：
```
场景：工单分类

提示词：
请判断用户描述的问题属于哪种类型：
- 故障报修：设备故障、停电、线路问题
- 业务咨询：电费查询、业务办理咨询
- 投诉建议：服务投诉、改进建议

只输出类型名称，不要输出其他内容。

标签：
- 故障报修
- 业务咨询
- 投诉建议
- 其他
```

#### 模块4：信息提取模块（6分钟）

**功能**：从输入信息中提取目标信息（关键词、参数等）

**界面展示**：
```
┌─────────────────────────────┐
│ 信息提取                     │
├─────────────────────────────┤
│ [模块名称] 信息提取          │
│                             │
│ 提示词：                     │
│ ┌─────────────────────────┐ │
│ │ 请从用户输入中提取关键信息│ │
│ └─────────────────────────┘ │
│                             │
│ 提取字段：                   │
│ ┌─────────────────────────┐ │
│ │ 字段key: region         │ │
│ │ 描述: 查询区域          │ │
│ │ 必填: ☑                 │ │
│ ├─────────────────────────┤ │
│ │ 字段key: time_range     │ │
│ │ 描述: 时间范围          │ │
│ │ 必填: ☐                 │ │
│ └─────────────────────────┘ │
│ + 添加字段                   │
├─────────────────────────────┤
│ 输出：                       │
│ ○ 完全提取（黄色）          │
│ ○ 必填完全提取（黄色）      │
│ ○ 提取缺失（黄色）          │
│ ○ 全部提取结果（蓝色）      │
│ ○ 模块运行结束（黄色）      │
└─────────────────────────────┘
```

**使用场景**：
- 提取搜索关键词
- 提取查询参数（时间、区域、指标）
- 配合HTTP调用使用

**案例**：
```
场景：用电数据查询

用户输入："查一下成都高新区这个月的用电量"

提取字段配置：
┌─────────────────────────────────────┐
│ 字段key: region                     │
│ 描述: 查询区域，如"成都高新区"       │
│ 必填: 是                            │
├─────────────────────────────────────┤
│ 字段key: time_range                 │
│ 描述: 时间范围，如"本月"、"最近7天"  │
│ 必填: 是                            │
├─────────────────────────────────────┤
│ 字段key: metric                     │
│ 描述: 查询指标，如"用电量"、"负荷"   │
│ 必填: 否                            │
└─────────────────────────────────────┘

提取结果：
{
  "region": "成都高新区",
  "time_range": "本月",
  "metric": "用电量"
}
```

#### 模块5：知识库搜索模块（6分钟）

**功能**：在知识库中搜索相关问题与解答

**界面展示**：
```
┌─────────────────────────────┐
│ 知识库搜索                   │
├─────────────────────────────┤
│ [模块名称] 知识库搜索        │
│                             │
│ 关联知识库：                 │
│ ☑ 电力安规知识库            │
│ ☑ 设备手册知识库            │
│ □ 故障案例知识库            │
│                             │
│ 相似度阈值：[0.7]           │
│ 向量相似度权重：[1.0]       │
│ 召回数：[5]                 │
│ □ 开启重排序                │
│ 重排序召回数：[5]           │
├─────────────────────────────┤
│ 输出：                       │
│ ○ 未搜索到相关知识（黄色）  │
│ ○ 搜索到相关知识（黄色）    │
│ ○ 知识库搜索结果（紫色）    │
│ ○ 模块运行结束（黄色）      │
└─────────────────────────────┘
```

**核心参数详解**：

| 参数 | 范围 | 建议 | 说明 |
|------|------|------|------|
| 相似度阈值 | 0-1 | 0.6-0.8 | 越高越严格，可能搜不到结果 |
| 向量相似度权重 | 0-1 | 1.0 | 有关键词时降低此值 |
| 召回数 | 0-100 | 5-10 | 返回多少条结果 |
| 重排序 | 是/否 | 精确度要求高时开启 | 会降低速度 |

**使用场景**：
- 知识库问答
- 文档检索
- FAQ问答

**案例**：
```
场景：电力安规问答

关联知识库：电力安规知识库
相似度阈值：0.7
召回数：5
重排序：开启

流程：
用户提问："变压器检修有哪些安全注意事项？"
     ↓
知识库搜索
     ↓
搜索到相关知识 → 智能对话（结合搜索结果回复）
未搜索到 → 确定回复（"抱歉，暂时无法回答"）
```

### Part 3：实操练习（15分钟）

#### 练习任务：创建一个知识库问答Agent

**步骤指南**：

```
步骤1：新建Agent
├── 点击"新建Agent"
├── 命名为"电力知识问答助手"
└── 设置描述和头像

步骤2：拖入模块
├── 拖入"用户提问"模块
├── 拖入"知识库搜索"模块
├── 拖入"智能对话"模块
└── 拖入"确定回复"模块

步骤3：连接模块
├── 用户提问 → 知识库搜索
│   - 文本信息 → 查询内容
│   - 模块运行结束 → 联动激活
│
├── 知识库搜索 → 智能对话（搜索到分支）
│   - 搜索到相关知识 → 联动激活
│   - 知识库搜索结果 → 知识库搜索结果
│
└── 知识库搜索 → 确定回复（未搜索到分支）
    - 未搜索到相关知识 → 联动激活
    - 设置回复内容："抱歉，暂时无法回答这个问题"

步骤4：配置知识库
├── 在知识库搜索模块中选择预置知识库
├── 设置相似度阈值：0.7
└── 设置召回数：5

步骤5：配置智能对话
├── 系统提示词：
│   "请根据知识库内容回答用户问题。
│    如果知识库内容与问题无关，
│    请说明无法回答。"
└── 回复创意性：0

步骤6：试运行测试
├── 点击"试运行"
├── 输入测试问题
├── 检查回复效果
└── 调整参数

步骤7：保存Agent
```

**讲师引导**：
- 学员跟随操作
- 巡场解答问题
- 收集常见问题
