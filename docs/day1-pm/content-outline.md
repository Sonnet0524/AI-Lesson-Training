# Day 1 下午课程大纲

## 课程信息

- **课程标题**: Agentic AI设计范式：RPAR认知架构与工程实现
- **总时长**: 120分钟（3课时 × 40分钟）
- **授课对象**: 国网四川电力AI种子团队
- **前置知识**: 上午已学习AI发展趋势和Agentic AI概念

---

## 课程目标

1. **理解Agent本质**: 掌握Agent与传统AI、Workflow的本质差别
2. **掌握RPAR框架**: 理解Reasoning-Planning-Acting-Reflection的认知循环
3. **了解范式演进**: 认识ReAct、Plan-and-Execute等RPAR的实例化
4. **理解工具演进**: 掌握API→MCP→Tools→Skills的技术栈演进

---

## 核心概念地图

```
Agentic AI = RPAR认知架构 + Skills能力封装 + 工具执行

                    ┌─────────────┐
                    │   Reasoning │ ← 思考：理解任务
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │   Planning  │ ← 规划：制定步骤
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │    Acting   │ ← 执行：调用工具
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  Reflection │ ← 反思：评估调整
                    └──────┬──────┘
                           └────────→ 回到Reasoning（循环）

工具层支持:
API(原始能力) → MCP(标准化) → Tools(原子能力) → Skills(语义封装)
```

---

## 三课时结构

### 课时1: Agent本质与RPAR框架 (40分钟)

**核心问题**: Agent的本质是什么？RPAR框架是什么？

**章节**:
1. **引入** (5分钟): 承接上午，提出核心问题
2. **Agent vs 传统AI** (10分钟): 执行能力的本质差别
3. **Workflow vs Agentic** (10分钟): 自主规划能力对比
4. **RPAR框架详解** (15分钟): 四环节的认知循环

**关键产出**:
- Agent = 执行能力 + 自主规划 + 反思调整
- RPAR循环图

---

### 课时2: RPAR实例与Acting详解 (40分钟)

**核心问题**: RPAR如何落地？Acting有哪些方式？

**章节**:
1. **RPAR的裁剪与实例化** (10分钟)
   - ReAct: R-A-R (简化版)
   - Plan-and-Execute: R-P-A (规划版)
   - 完整RPAR: R-P-A-R (完全体)
   
2. **范式对比与选择** (10分钟)
   - 三种范式的适用场景
   - 如何选择合适的范式
   
3. **Acting详解** (20分钟)
   - API调用
   - 本地代码执行
   - 文件系统操作
   - 其他工具调用

**关键产出**:
- 范式选择决策树
- Acting四象限图

---

### 课时3: 工具演进与封装 (40分钟)

**核心问题**: 工具如何演进？如何封装？

**章节**:
1. **API → MCP** (10分钟)
   - API的局限：AI看不懂
   - MCP的突破：自描述能力
   - 从"人能懂"到"AI能懂"
   
2. **Tools → Code Tools → Skills** (15分钟)
   - Tools: 原子能力
   - Code Tools: 自定义逻辑
   - Skills: 业务语义封装
   
3. **完整技术栈架构** (15分钟)
   - 从API到RPAR的完整层次
   - 各层职责和关系
   - 灵知平台的映射

**关键产出**:
- 工具演进阶梯图
- 完整架构层次图
