# Day 1 下午课程大纲

## 课程信息

- **课程标题**: Agentic AI设计范式：RPAR认知架构与工程实现
- **总时长**: 120分钟（3课时 × 40分钟）
- **授课对象**: 国网四川电力AI种子团队
- **前置知识**: 上午已学习AI发展趋势和Agentic AI概念
- **目标字数**: 25,000字
- **当前版本**: v2.0（已扩充）

---

## 课程目标

1. **理解Agent本质**: 掌握Agent与传统AI、Workflow的本质差别
2. **掌握RPAR框架**: 理解Reasoning-Planning-Acting-Reflection的认知循环
3. **了解范式演进**: 认识ReAct、Plan-and-Execute等RPAR的实例化
4. **理解工具演进**: 掌握API→MCP→Tools→Skills的技术栈演进
5. **建立完整认知**: 构建Agentic AI的六层技术栈认知体系

---

## 核心概念地图

```
Agentic AI = RPAR认知架构 + Skills能力封装 + 工具执行

                    ┌─────────────┐
                    │   Reasoning │ ← 思考：理解任务
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │   Planning  │ ← 规划：制定步骤
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │    Acting   │ ← 执行：调用工具
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  Reflection │ ← 反思：评估调整
                    └──────┬──────┘
                           └────────→ 回到Reasoning（循环）

工具层支持:
API(原始能力) → MCP(标准化) → Tools(原子能力) → Skills(语义封装)

完整六层架构:
API → MCP → Tools → Code Tools → Skills → RPAR
```

## 三课时结构

### 课时1: Agent本质与RPAR框架 (40分钟) - 约8,200字

**核心问题**: Agent的本质是什么？RPAR框架是什么？

**章节**:
1. **引入** (15分钟): 承接上午，提出核心问题
   - 开场引导问题（5个引发思考的问题）
   - 学员互动设计（举手表决、小组讨论）
   - 课程目标可视化图表
   - 电力行业价值说明（4个痛点分析）
   
2. **Agent vs 传统AI** (10分钟): 执行能力的本质差别
   - 传统大模型应用特点（交互模式、能力范围、完成度）
   - Agent核心能力（执行+规划+反思）
   - 详细对比表
   - 3个电力行业对比案例（查询用电数据、故障诊断、报表生成）
   - 执行能力技术原理解释
   
3. **Workflow vs Agentic** (10分钟): 自主规划能力对比
   - Workflow特点与局限
   - Agentic特点与优势
   - 3个类比说明（高铁vs自驾、菜谱vs厨师、导航vs自动驾驶）
   - 决策树：什么时候用Workflow，什么时候用Agentic
   
4. **RPAR框架详解** (5分钟): 四环节的认知循环
   - Reasoning（思考）5个维度详解
   - Planning（规划）3种策略详解
   - Acting（执行）工具选择逻辑
   - Reflection（反思）3种类型详解
   - 电力行业投资分析案例走3轮RPAR循环

**关键产出**:
- Agent = 执行能力 + 自主规划 + 反思调整
- RPAR循环图（详细版）
- 范式选择决策树
- 5个电力行业实战案例

     - 文件读写完整流程

     - 批量文件处理场景
     - 存储路径和权限管理
   - 其他工具详解
     - 浏览器自动化（Selenium原理）

## 讲师要点提示

### 开场钩子
> "上午专家讲了Agent的历史，下午我们解决一个核心问题：Agent与传统AI应用的本质差别是什么？答案只有一个字——'干'。Agent不只是说，而是真干。"

### 三个金句
1. **Agent本质**: "从'动口'到'动手'，从'顾问'到'助手'。"
2. **RPAR核心**: "ReAct是RPAR的简化版，RPAR是Agent的'完全体'。"
3. **工具演进**: "从'人能懂'到'AI能懂'，MCP让工具具备了'自我介绍'能力。"

### 新增金句
4. **范式选择**: "没有最好的范式，只有最适合的范式。"
5. **MCP价值**: "MCP不是'能不能'的问题，是'通不通'的问题。"
6. **Skills本质**: "Tools解决'怎么做'，Skills解决'做什么'。"

### 板书重点
- **必须画图1**: RPAR循环图 (四象限详细版)
- **必须画图2**: Acting四象限 (2x2网格)
- **必须画图3**: 工具演进阶梯 (API→MCP→Tools→Skills→RPAR)
- **必须画图4**: 六层架构图
- **必须画图5**: 范式选择决策树
