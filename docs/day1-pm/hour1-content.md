## 2. Agent vs 传统AI (10分钟)

### 讲解要点

**传统大模型应用**:
- **交互模式**: 一问一答
- **能力范围**: 只给建议，不执行
- **完成度**: 告诉你怎么做，但不帮你做

**Agent**:
- **交互模式**: 多轮自主交互
- **能力范围**: 直接执行操作
- **完成度**: 直接帮你做完

**对比表**:

| 维度 | 传统AI应用 | Agent |
|------|-----------|-------|
| **交互** | 单轮对话 | 多轮自主 |
| **执行** | ❌ 只给建议 | ✅ 直接执行 |
| **完成度** | "你应该..." | "已完成" |
| **价值** | 信息 | 结果 |

### 核心观点

> **"Agent的本质是'执行'，特别是'自主执行'——从'动口'到'动手'。"**

### 讲师话术示例

> "想象一下，你问传统AI：'帮我查一下成都高新区的用电数据。'它会告诉你：'你可以登录系统，点击查询，选择区域...'它只告诉你怎么做。但Agent不一样，Agent会直接登录系统、查询数据、整理结果，最后把报告放在你面前。这就是本质差别：从'顾问'到'助手'，从'动口'到'动手'。"

### 白板/板书建议

**画对比**:
```
传统AI:  提问 → 回答建议
              ↓
         [人执行]

Agent:   提问 → 自主执行 → 完成结果
```

## 3. Workflow vs Agentic (10分钟)

### 讲解要点

**Workflow（工作流）**:
- ✅ 有执行能力：按预设步骤执行
- ❌ 无自主规划：步骤是固定的
- ❌ 无反思调整：走错就失败

**Agentic（智能体）**:
- ✅ 执行能力
- ✅ **自主规划**（根据任务动态制定计划）
- ✅ **反思调整**（根据反馈优化下一步）

**类比说明**:

| 比喻 | Workflow | Agentic |
|------|----------|---------|
| **交通工具** | 高铁 | 自驾 |
| **特点** | 速度快但路线固定 | 可根据路况调整路线 |
| **能力** | 按既定路线执行 | 自主规划路线 |

**关键洞察**:
> "Workflow是'提线木偶'，Agent是'自主机器人'。差别就在**自主规划**和**反思**能力。"

### 讲师话术示例

> "那有人说，我们的系统也有执行能力啊，Workflow工作流不就是吗？工作流确实能执行，但它是'提线木偶'——每一步都是预设好的，如果中间出错就卡住了，它不会自己想办法。而Agent是'自主机器人'，它能看路、能判断、能调整。比如工作流走到某一步发现数据不对，它就卡住了；但Agent会反思：这个数据不对，我应该换个方式获取，或者我应该向用户确认。这就是自主规划和反思的能力。"

### 白板/板书建议

**画图对比**:
```
Workflow (高铁):
A → B → C → D
└──── 固定路线 ────┘
   走错就停止

Agentic (自驾):
    ┌→ B →┐
起点 ─┤     ├→ 终点
    └→ C →┘
   实时调整路线
```

## 4. RPAR框架详解 (15分钟)

### 讲解要点

**引出RPAR**:
> "既然Agent的核心是'自主执行'，那它是如何自主的？答案就是RPAR框架。"

**RPAR四环节**:

| 环节 | 英文 | 核心问题 | 人类类比 |
|------|------|----------|---------|
| **R-思考** | Reasoning | "我现在面对什么问题？" | 观察分析 |
| **P-规划** | Planning | "我需要分几步做？" | 制定计划 |
| **A-执行** | Acting | "调用什么工具？" | 动手执行 |
| **R-反思** | Reflection | "这样做对吗？" | 复盘调整 |

**循环特性**:
```
思考 → 规划 → 执行 → 反思
  ↑___________________|
        持续迭代
```

**与人类的对比**:

人类解决问题:
1. 看到问题 → 思考分析
2. 制定方案 → 规划步骤
3. 动手实施 → 执行操作
4. 检查效果 → 反思调整
5. 回到1，直到完成

Agent的RPAR就是模拟这个过程。

### 核心观点

**三个突破**:
1. **从黑盒到白盒**: 显式推理过程，可审计
2. **从静态到动态**: 与环境持续交互，边做边学
3. **从确定到适应**: 容错性强，可自我纠正

### 讲师话术示例

**开场**:
> "RPAR，四个字母，四个环节：Reasoning思考、Planning规划、Acting执行、Reflection反思。这四个环节构成一个循环，Agent就是通过这个循环来自主解决问题的。"

**展开讲解**:
> "第一环节，Reasoning思考。Agent首先要理解任务，分析现状。就像你接到一个任务，第一反应是：这是什么问题？我需要什么信息？第二环节，Planning规划。想清楚之后，Agent要制定计划：分几步做？先做哪步？第三环节，Acting执行。有了计划就动手，调用工具去执行。第四环节，Reflection反思。执行完了要检查：结果对吗？达到目标了吗？如果没有，回到第一步重新思考。这就是RPAR循环。"

**强调循环**:
> "注意，这不是线性的，是循环的。反思之后可能发现做错了，那就回到思考环节重新来。就像你做数学题，算错了要重新审题、重新列式子。Agent也是这样，它有试错和修正的能力。"

### 白板/板书建议

**画核心图 - RPAR循环**:
```
                    ┌──────────────────┐
                    │    Reasoning     │
                    │     (思考)       │
                    │  理解任务现状    │
                    └────────┬─────────┘
                             ↓
                    ┌──────────────────┐
                    │    Planning      │
                    │     (规划)       │
                    │  制定执行步骤    │
                    └────────┬─────────┘
                             ↓
                    ┌──────────────────┐
                    │     Acting       │
                    │     (执行)       │
                    │  调用工具执行    │
                    └────────┬─────────┘
                             ↓
                    ┌──────────────────┐
                    │   Reflection     │
                    │     (反思)       │
                    │  评估调整策略    │
                    └────────┬─────────┘
                             └──────────────→ 
                                 回到Reasoning
```

## 课时小结

### 核心要点回顾

1. **Agent本质**: 执行能力 + 自主规划 + 反思调整
2. **与传统AI差别**: 从"动口"到"动手"
3. **与Workflow差别**: 从"固定流程"到"自主规划"
4. **RPAR框架**: Reasoning-Planning-Acting-Reflection循环

### 金句

> "Agent不是更聪明的问答系统，而是能自主执行任务的智能体。RPAR是这个智能体的'认知架构'——它模拟了人类解决问题的思维方式。"
