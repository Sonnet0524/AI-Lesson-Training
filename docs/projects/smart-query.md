# 项目设计：智能问数

## 项目概述

**项目名称**：电力数据智能问答系统

**项目目标**：通过自然语言对话方式查询电力数据，自动生成数据分析和可视化图表

**目标用户**：电力公司管理人员、数据分析人员、运维人员

**复杂度**：⭐⭐⭐

---

## 一、业务场景

### 典型需求

```
场景1：领导查询
"查一下本月各区域的用电量排名"

场景2：趋势分析
"对比一下成都高新区最近7天的负荷变化趋势"

场景3：异常排查
"昨天半夜负荷突然升高是什么原因？"

场景4：数据对比
"今年一季度和去年同期相比，哪些区域增长最快？"
```

### 用户痛点

- 传统方式：需要登录多个系统、手写SQL、手动制图
- 智能问数：自然语言提问 → 自动查询 → 自动分析 → 自动可视化

### Agent流程设计（ReAct模式）

```
用户提问
    ↓
【Thought】分析用户意图
"用户想查询用电量数据，需要识别时间范围、区域、指标"
    ↓
【Action】调用信息提取
提取参数：{
  "metric": "用电量",
  "region": "各区域",
  "time_range": "本月",
  "operation": "排名"
}
    ↓
【Observation】获取参数成功
    ↓
【Thought】确定查询方案
"需要查询本月各区域用电量，然后排序"
    ↓
【Action】调用数据查询API
query_power_data(metric="用电量", region="all", time_range="本月")
    ↓
【Observation】获取数据
[
  {"region": "高新区", "value": 850},
  {"region": "天府新区", "value": 820},
  ...
]
    ↓
【Thought】数据处理和可视化
"数据已获取，需要排序并生成图表"
    ↓
【Action】生成分析结果和图表
    ↓
【Final Answer】返回结果
"本月各区域用电量排名如下：
1. 高新区：850万kWh
2. 天府新区：820万kWh
3. 锦江区：780万kWh
[附带柱状图]"
```

## 三、灵知平台模块配置

### 模块流程图

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 用户提问 │────→│ 信息提取 │────→│ HTTP调用 │────→│ 信息加工 │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                                                        │
                  ┌──────────┐                          │
                  │ 智能对话 │←─────────────────────────┘
                  └──────────┘
```

### 模块详细配置

#### 1. 用户提问模块

```
模块名称：用户提问
配置：
- ☑ 输入文本
- ☐ 上传文档
- ☐ 上传图片
- ☐ 添加按钮

输出：
- 文本信息 → 信息提取模块
```

#### 2. 信息提取模块

```
模块名称：查询参数提取
提示词：
请从用户问题中提取以下信息：
- metric：查询指标（用电量/负荷/电量/功率）
- region：查询区域（具体区域名或"全部"）
- time_range：时间范围（如"本月"、"最近7天"、"2024年1月"）
- operation：操作类型（排名/对比/趋势/明细）

如果用户没有明确说明某项，请根据上下文推断或使用默认值。

输出JSON格式，如：
{
  "metric": "用电量",
  "region": "全部",
  "time_range": "本月",
  "operation": "排名"
}

提取字段配置：
┌─────────────────────────────────────┐
│ 字段key: metric                     │
│ 描述: 查询指标                      │
│ 必填: 是                            │
├─────────────────────────────────────┤
│ 字段key: region                     │
│ 描述: 查询区域                      │
│ 必填: 否（默认全部）                 │
├─────────────────────────────────────┤
│ 字段key: time_range                 │
│ 描述: 时间范围                      │
│ 必填: 否（默认本月）                 │
├─────────────────────────────────────┤
│ 字段key: operation                  │
│ 描述: 操作类型                      │
│ 必填: 否（默认明细）                 │
└─────────────────────────────────────┘

回复创意性：0
```

#### 3. HTTP调用模块

```
模块名称：数据查询
调用方式：POST
URL：https://api.power.com/query
Headers：
  Content-Type: application/json
  Authorization: Bearer {token}

请求体模板：
{
  "metric": "{{信息提取.metric}}",
  "region": "{{信息提取.region}}",
  "time_range": "{{信息提取.time_range}}",
  "operation": "{{信息提取.operation}}"
}

输出：
- 响应数据 → 信息加工模块
```

#### 4. 信息加工模块

```
模块名称：数据格式化
类型：代码块（JavaScript）

function userFunction(param) {
  const data = JSON.parse(param['response']);
  
  // 排序
  if (data.operation === '排名') {
    data.results.sort((a, b) => b.value - a.value);
  }
  
  // 格式化输出
  let result = {
    summary: `共查询到${data.results.length}条数据`,
    data: data.results,
    chart_type: data.operation === '趋势' ? 'line' : 'bar'
  };
  
  return result;
}

输出：
- 格式化数据 → 智能对话模块
```

#### 5. 智能对话模块

```
模块名称：分析回复
系统提示词：
你是电力数据分析助手，负责解读查询结果并以清晰的方式呈现给用户。

当前查询结果：
{{信息加工结果}}

请按照以下格式回复：

📊 查询结果

[简要说明查询内容和结果]

📈 数据明细
1. [项目1]：[数值] [单位]
2. [项目2]：[数值] [单位]
...

💡 分析建议
[根据数据给出简单的分析或建议]

回复创意性：0.3
聊天上下文：3
回复字数上限：500
```

## 五、API接口设计

### 数据查询接口

**请求**：
```json
POST /api/query
{
  "metric": "用电量",
  "region": "全部",
  "time_range": "本月",
  "operation": "排名"
}
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "metric": "用电量",
    "unit": "万kWh",
    "time_range": "2024年3月",
    "results": [
```
