# 基础配置

## 配置概述

智能体的配置分为 **3 大部分**：

1. **基本信息配置**：头像、名称、开场白等
2. **规划配置**：模块化编排实现功能
3. **分发配置**：删除、保存、发布等操作

本页面重点介绍 **基本信息配置**。

---

## 创建准备（5步骤）

在创建智能体之前，建议完成以下准备工作：

### 步骤 1：评估当前需求

**明确**：
- 业务场景的目的和目标
- 目标用户群体
- 核心功能需求

**示例**：
```
场景：智能客服
目的：自动回答用户常见问题
目标用户：网站访客
核心功能：FAQ问答、工单查询、人工转接
```

---

### 步骤 2：评估AI能力

**匹配需求与平台能力**：

| 需求 | 平台能力 | 适用模块 |
|------|----------|----------|
| 问答对话 | ✅ 智能对话 | 智能对话模块 |
| 知识查询 | ✅ 知识库搜索 | 知识库搜索模块 |
| 意图识别 | ✅ 信息分类 | 信息分类模块 |
| 数据提取 | ✅ 信息提取 | 信息提取模块 |
| 文档处理 | ✅ 文档问答 | 文档提问模块 |

---

### 步骤 3：画布编排

**设计流程**：
- 使用纸笔画出流程图
- 确定关键模块和数据流向
- 考虑异常情况和兜底方案

---

### 步骤 4：用户交互设计

**设计要素**：
- 对话引导语
- 错误提示语
- 用户操作引导
- 退出机制

---

### 步骤 5：测试与实验

**测试清单**：
- ✅ 正常场景测试
- ✅ 异常输入测试
- ✅ 边界情况测试
- ✅ 性能压力测试

---

## 基本信息

### 智能体头像

**选项**：
1. **自定义上传**：上传图片（推荐尺寸 128x128px）
2. **文字头像**：输入文字生成头像
3. **平台头像**：选择平台预设头像

**建议**：
- 使用与业务相关的图标
- 保持风格统一
- 避免使用过于复杂的图片

---

### 智能体名称

**要求**：
- 必填项
- 长度：2-20 字符
- 清晰描述用途

**命名建议**：
- ✅ "智能客服助手"
- ✅ "订单查询机器人"
- ✅ "技术支持助手"
- ❌ "机器人1"
- ❌ "助手"

---

### 智能体描述

**要求**：
- 选填项
- 长度：0-200 字符
- 详细说明功能

**描述建议**：
```
这是一个智能客服助手，可以：
1. 回答常见问题
2. 查询订单状态
3. 处理工单提交
4. 转接人工客服

适用场景：网站客服、在线咨询
```

---

### 对话背景

**选项**：
1. **默认白色**：简洁干净
2. **精美背景**：平台提供的背景图
3. **自定义上传**：上传背景图片

**建议**：
- 选择浅色背景（文字更清晰）
- 避免过于花哨的背景
- 保持与品牌风格一致

---

## 开场设置

### 开场白

**作用**：对话开始前，用户首先看到的消息

**要求**：
- 必填项
- 长度：0-500 字符
- 友好、简洁

**开场白模板**：

**简洁型**：
```
您好！我是智能助手，请问有什么可以帮助您的？
```

**功能型**：
```
👋 您好！我是智能客服助手

我可以帮您：
✅ 查询订单状态
✅ 回答常见问题
✅ 处理售后问题
✅ 转接人工客服

请问有什么可以帮助您的？
```

**引导型**：
```
欢迎使用智能助手！

请选择您需要的服务：
1️⃣ 订单查询
2️⃣ 产品咨询
3️⃣ 售后服务
4️⃣ 人工客服

请输入序号或直接描述您的需求。
```

---

### 问题建议

**作用**：预设问题引导，用户可以直接点击

**配置**：
- 最多添加 6 个问题
- 在首次对话时呈现
- 引导用户快速开始

**问题建议模板**：

**客服场景**：
```
1. 你能做什么？
2. 如何查询订单？
3. 退货流程是什么？
4. 如何联系人工客服？
```

**查询场景**：
```
1. 查询本月销售数据
2. 生成周报
3. 导出数据报表
4. 查看趋势分析
```

**咨询场景**：
```
1. 产品价格是多少？
2. 有什么优惠政策？
3. 如何开通服务？
4. 技术支持联系方式
```

## 模型配置

### 对话模型

**作用**：设定默认"智能对话"模块所使用的模型

**可选模型**：
- Qwen-Plus（推荐）
- GPT-4
- GLM-4
- 其他模型

**选择建议**：
- **Qwen-Plus**：中文场景，效果好
- **GPT-4**：英文场景，创意性强
- **GLM-4**：平衡性价比

---

### 提示词

**作用**：设定默认的系统提示词

**提示词模板**：

**客服场景**：
```markdown
# 角色
你是一个专业的客服助手。

# 职责
- 回答用户关于产品和服务的疑问
- 处理订单查询和售后服务
- 提供友好的用户体验

# 要求
- 态度友好、专业
- 回答准确、简洁
- 遇到不确定的问题，诚实说明并转接人工
```

**数据分析场景**：
```markdown
# 角色
你是一个数据分析助手。

# 能力
- 查询和分析业务数据
- 生成统计报告
- 提供数据洞察

# 要求
- 数据准确无误
- 分析客观理性
- 结论清晰易懂
```

---

### 知识库

**作用**：设定默认"知识库搜索"模块所关联的知识库

**配置**：
- 单个智能体可选择多个知识库
- 对话过程中将对知识库进行内容检索

**建议**：
- 选择相关的知识库
- 避免选择过多无关知识库（影响检索精度）

---

### 是否显示知识库引用

**作用**：通过知识库搜索回复的答案，是否展示参考信息的文档来源

**选项**：
- ✅ 开启：显示引用来源
- ❌ 关闭：不显示引用来源

**建议**：
- 专业场景：开启，增加可信度
- 轻松场景：关闭，提升体验

## 配置示例

### 示例 1：智能客服

**基本信息**：
```
智能体名称：智能客服助手
智能体描述：为用户提供产品咨询、订单查询、售后服务等支持
头像：客服图标
```

**开场设置**：
```
开场白：
👋 您好！我是智能客服助手

我可以帮您：
✅ 产品咨询
✅ 订单查询
✅ 售后服务
✅ 人工转接

请问有什么可以帮助您的？

问题建议：
1. 你能做什么？
2. 如何查询订单？
3. 退货流程是什么？
4. 联系人工客服
```

---

### 示例 2：数据分析助手

**基本信息**：
```
智能体名称：数据分析助手
智能体描述：帮助用户查询业务数据、生成分析报告、提供数据洞察
头像：数据分析图标

```
开场白：

🔍 数据趋势分析

---

### 示例 3：文档问答助手

**基本信息**：
```
智能体名称：文档问答助手
智能体描述：帮助用户理解文档内容、回答文档相关问题
头像：文档图标
```

**开场设置**：
```
开场白：
📄 您好！我是文档问答助手

请上传您的文档（PDF、Word、TXT），我将为您：
✅ 总结文档内容
✅ 回答相关问题
✅ 提取关键信息
✅ 分析文档结构

请上传文档并告诉我您的问题。

问题建议：
1. 这份文档的主要内容是什么？
2. 文档中有哪些关键数据？
3. 总结文档的核心观点
4. 文档的结构是什么？
```

## 最佳实践

### 1. 开场白设计

✅ **推荐**：
- 简洁友好
- 明确功能
- 提供引导

❌ **避免**：
- 过于冗长
- 功能不清晰
- 缺少引导

---

### 2. 问题建议设计

✅ **推荐**：
- 覆盖主要功能
- 描述清晰具体
- 易于理解

❌ **避免**：
- 问题过于宽泛
- 数量过多（>6个）
- 描述模糊

---

### 3. 提示词设计

✅ **推荐**：
- 明确角色定位
- 清晰职责范围
- 具体要求说明

❌ **避免**：
- 角色定义模糊
- 职责不清晰
- 要求过于抽象
