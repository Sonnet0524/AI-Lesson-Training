#### Part 2：规划页面与模块架构（20分钟）

**界面三大区域**：
```
┌─────────────────────────────────────────────────────────────┐
│                      功能区                                  │
│  [导入/导出] [试运行]           [分享] [删除] [保存] [发布]  │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│ 模块区   │                  画布区                           │
│          │                                                  │
│ - 系统模块│    （拖拽组件、连线编排的区域）                   │
│ - 智能体 │                                                  │
│ - 工具   │                                                  │
│ - MCP   │                                                  │
│          │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

**模块区4类组件**：
- **系统模块**：用户提问、智能对话、信息分类、信息提取、信息加工、确定回复、知识库搜索等
- **智能体**：调用已上线的其他智能体（子智能体）
- **工具**：官方工具 + 自定义工具
- **MCP**：连接外部服务（SSE方式）

**节点类型与连接原则**：
| 颜色 | 类型 | 连接规则 |
|------|------|----------|
| 黄色 | 布尔型 | 只能连布尔型 |
| 蓝色 | 字符串 | 文本/图片/文档信息 |
| 紫色 | 知识库结果 | 仅连知识库相关 |
| 红色 | 任意类型 | 仅用于循环模块 |

**模块结构**：
```
┌─────────────────────────┐
│  模块输入区（左上角）    │  ← 连接节点将信息输入
│  - 联动激活              │
│  - 任一激活              │
│  - 信息输入              │
├─────────────────────────┤
│  模块设置区（中间）      │  ← 设置参数、提示词
│  - 聊天上下文            │
│  - 模型选择              │
│  - 系统提示词            │
│  - 回复创意性            │
├─────────────────────────┤
│  模块输出区（右下角）    │  ← 执行后输出信息
│  - 回复结束              │
│  - 回复内容              │
│  - 模块运行结束          │
└─────────────────────────┘
```

#### Part 3：核心模块详解（30分钟）

**模块1：用户提问模块**

**功能**：获取用户提问/选择内容，传输给其他模块

**核心参数**：
- 输入文本（默认打开）
- 上传文档（最多9个，每个50MB）
- 上传图片（最多9个，单个30M）
- 文档审查（配合文档审查模块）
- 添加按钮（明确执行节点）

**使用场景**：所有Agent的入口模块

---

**模块2：智能对话模块**

**功能**：通过大语言模型处理并回复用户

**核心参数**：
| 参数 | 建议值 | 说明 |
|------|--------|------|
| 聊天上下文 | 0-6条 | 贴近用户意图 |
| 回复创意性 | 0-1 | 0严谨，1发散（知识库场景建议0） |
| 核采样TOP_P | 0-1 | 控制多样性，一般与创意性只设一个 |
| 回复字数上限 | 100-8192 | 根据场景设定 |
| 回复对用户可见 | 是/否 | 中间流程可设为否 |

**使用场景**：所有需要AI回复的场景

---

**模块3：信息分类模块**

**功能**：将用户问题分类，针对不同类型执行不同操作

**核心参数**：
- 提示词：说明分类任务
- 标签：设置分类类别（如"咨询""投诉""建议""其他"）

**输出**：
- 标签节点（布尔型）：符合该分类输出True
- 分类结果（JSON格式）

**使用场景**：工单分类、问题类型识别、多流程分支

---

**模块4：信息提取模块**

**功能**：从输入信息中提取目标信息（关键词、SQL等）

**核心参数**：
- 提示词：说明提取任务
- 提取字段：配置字段key、描述、是否必填

**输出**：
- 完全提取/必填完全提取/提取缺失（布尔型）
- 全部提取结果（JSON字符串）

**使用场景**：提取搜索关键词、提取结构化数据、配合HTTP调用

---

**模块5：知识库搜索模块**

**功能**：在知识库中搜索相关问题与解答

**核心参数**：
| 参数 | 范围 | 建议 |
|------|------|------|
| 相似度阈值 | 0-1 | 0.6-0.9（越高越严格） |
| 向量相似度权重 | 0-1 | 默认1，有关键词时降低 |
| 召回数 | 0-100 | 建议5-10 |
| 是否开启重排序 | 是/否 | 提高精确度但降低速度 |

**使用场景**：知识库问答、文档检索、客服问答

---

#### Part 4：常见编排模式（10分钟）

**模式1：简单问答**
```
用户提问 → 智能对话 → 回复用户
```

**模式2：知识库问答**
```
用户提问 → 知识库搜索 → 
  ├─ 搜索到 → 智能对话（结合知识库） → 回复
  └─ 未搜索到 → 确定回复（默认回答）
```

**模式3：分类处理**
```
用户提问 → 信息分类 → 
  ├─ 类型A → 处理流程A → 回复
  ├─ 类型B → 处理流程B → 回复
  └─ 其他 → 引导重新描述
```

**模式4：提取+API调用**
```
用户提问 → 信息提取 → HTTP调用 → 信息加工 → 回复
```

#### Part 5：实操练习（15分钟）

**练习任务**：创建一个知识库问答Agent

1. 进入"规划"页面
2. 拖入"用户提问"模块
3. 拖入"知识库搜索"模块
4. 拖入"智能对话"模块
5. 连接模块：用户提问 → 知识库搜索 → 智能对话
6. 配置知识库（使用预置知识库）
7. 点击"试运行"测试
8. 保存Agent

#### Part 6：答疑与作业布置（10分钟）
- 常见问题解答
- 预习作业布置

**预期产出**：
- ✅ 学员完成灵知平台账号准备
- ✅ 理解规划页面和模块架构
- ✅ 掌握5个核心模块使用
- ✅ 能够独立创建简单Agent

## 课程安排建议

### 方案A：单次直播（推荐）

**时长**：2小时

**时间安排**：
- 0:00-0:15 平台介绍与账号准备

- 0:35-1:05 核心模块详解
